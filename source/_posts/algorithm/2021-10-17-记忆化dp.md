#### leetcode 496 预测赢家

```
给你一个整数数组 nums 。玩家 1 和玩家 2 基于这个数组设计了一个游戏。

玩家 1 和玩家 2 轮流进行自己的回合，玩家 1 先手。开始时，两个玩家的初始分值都是 0 。每一回合，玩家从数组的任意一端取一个数字(即，nums[0] 或 nums[nums.length - 1])，取到的数字将会从数组中移除（数组长度减 1 ）。玩家选中的数字将会加到他的得分上。当数组中没有剩余数字可取时，游戏结束。

如果玩家 1 能成为赢家，返回 true 。如果两个玩家得分相等，同样认为玩家 1 是游戏的赢家，也返回 true 。你可以假设每个玩家的玩法都会使他的分数最大化。

1 <= nums.length <= 20
0 <= nums[i] <= 10^7
```

这是个零和博弈, 无论采取何种策略，博弈双方的总得分不变。即一个人得分变多，另一个必然减少。

零和博弈问题的关键都是要找出如何让后手更差的局面的方法。让后手更差其实就是让先手更优。

零和博弈自己得分多和对手得分少同时存在的，因为总分一致。可以设置`dp[L][R]`为在`[L，R]`段的最佳选择，也就是能得到最高的分。而选择之后对手为`dp[L+1][R]`，要么是`dp[L][R-1]`。因此此步目标是`min(dp[L+1][R], dp[L][R-1])`使对手得分最小, 此时先手得分为`sum(L, R)-min(dp[L+1][R], dp[L][R-1])`

可以看出子串`[L,R]`可能被重复计算多次，因此可用动态规划求解。

```cpp
class Solution {
public:
    int dp[21][21];
    int sum(const vector<int> &nums, int L, int R) {
        int sum = 0;
        for(int i = L; i <= R; i++) {
            sum += nums[i-1];
        }
        return sum;
    }
    // dfs巧妙地在于, 每个回合就是一个dfs递归栈, 在这个回合双方都会选最好的选择, 即 sum(nums, L, R) - min(dfs(L+1, R, nums), dfs(L, R-1, nums));, 而玩家一的优势只有先手这一个
    int dfs(int L, int R, const vector<int> &nums) {
        if(dp[L][R] != -1) {    /// 记忆化搜索
            return dp[L][R];
        }
        /// 没得选
        if(L == R) {
            return dp[L][R] = nums[L-1];
        }
        /// min用得巧妙
        return dp[L][R] = sum(nums, L, R) - min(dfs(L+1, R, nums), dfs(L, R-1, nums));
    }
    bool PredictTheWinner(vector<int>& nums) {
        memset(dp, -1, sizeof(dp));
        /// 得分是否过半, 如果得分过半表示在dp[1][n]条件下先手可以通过选择使后手肯定得分小于先手
        //dp[L][R]表示数组的L,R段先手可选的最大值
        return dfs(1, nums.size(), nums)*2 >= sum(nums, 1, nums.size());
    }
};
```

#### 464 我能赢吗

```
在 "100 game" 这个游戏中，两名玩家轮流选择从 1 到 10 的任意整数，累计整数和，先使得累计整数和 达到或超过  100 的玩家，即为胜者。

如果我们将游戏规则改为 “玩家 不能 重复使用整数” 呢？

例如，两个玩家可以轮流从公共整数池中抽取从 1 到 15 的整数（不放回），直到累计整数和 >= 100。

给定两个整数 maxChoosableInteger （整数池中可选择的最大数）和 desiredTotal（累计和），若先出手的玩家是否能稳赢则返回 true ，否则返回 false 。假设两位玩家游戏时都表现 最佳 。

输入：maxChoosableInteger = 10, desiredTotal = 11
输出：false
```

同样类似上题的博弈, 只是选择的数字不能重复。

特判：

* 如果最大能选择的数字maxChoosableInteger比期望的总数desiredTotal要大，先手稳赢，返回True
* 如果能选择的所有数字总和比期望的总数desiredTotal要小，一定到达不了desiredTotal，先手稳输，返回False(因为不能选择重复数字)

状态, 对于n个数字一共有2^n个状态, 因此记忆化数组需要开辟2^n的空间, dp[state]表示在state下当前先手能否赢。

```py
class Solution:
    def canWin(self, maxChoosableInteger:int, desiredTotal: int) -> bool :
        if maxChoosableInteger >= desiredTotal:
            return True 
        if sum(range(1,maxChoosableInteger+1)) < desiredTotal:
            return False
        
        def dfs(state, desiredTotal, dp):
            if dp[state] != None:
                return dp[state]
            
            for i in range(1, maxChoosableInteger+1):
                cur = 1 << (i-1)
                if cur & state != 0:
                    continue
                if i >= desiredTotal or not dfs(state | cur, desiredTotal-i, dp):   # 对手输, i>=desiredTotal这里很重要, 相当于自底而上的底
                    dp[state] = True 
                    return True 
            dp[state] = False # 在当前state下先手只有输
            return False

        return dfs(0, desiredTotal, [None] * (1 << maxChoosableInteger))
        
```


#### 太平洋大西洋水流问题

```
给定一个 m x n 的非负整数矩阵来表示一片大陆上各个单元格的高度。"太平洋"处于大陆的左边界和上边界，而"大西洋"处于大陆的右边界和下边界。

规定水流只能按照上、下、左、右四个方向流动，且只能从高到低或者在同等高度上流动。

请找出那些水流既可以流动到"太平洋"，又能流动到"大西洋"的陆地单元的坐标。

输出坐标的顺序不重要
m 和 n 都小于150

给定下面的 5x5 矩阵:

  太平洋 ~   ~   ~   ~   ~ 
       ~  1   2   2   3  (5) *
       ~  3   2   3  (4) (4) *
       ~  2   4  (5)  3   1  *
       ~ (6) (7)  1   4   5  *
       ~ (5)  1   1   2   4  *
          *   *   *   *   * 大西洋

返回:

[[0, 4], [1, 3], [1, 4], [2, 2], [3, 0], [3, 1], [4, 0]] (上图中带括号的单元).
```

可以从左上边界每个坐标开始dfs,找到可以流到太平洋的坐标

再从右下边界dfs,找到可以流到大西洋的坐标

二者坐标的交集就是所求

```cpp
class Solution {
public:
    vector<vector<int>> P, A, ans;
    int n, m;
    vector<vector<int>> pacificAtlantic(vector<vector<int>>& M) {
        n = M.size(), m = M[0].size();
        /// P,A分别记录大西洋，太平洋可流经的地点
        P = A = vector<vector<int>>(n, vector<int>(m, 0));
        //左右两边加上下两边出发深搜
        for(int i = 0; i < n; ++i) dfs(M, P, i, 0), dfs(M, A, i, m - 1);
        for(int j = 0; j < m; ++j) dfs(M, P, 0, j), dfs(M, A, n - 1, j);             
        return ans;
    }
    void dfs(vector<vector<int>>& M, vector<vector<int>>& visited, int i, int j){        
        if(visited[i][j]) return;
        visited[i][j] = 1;  /// visited可以是P或者A的形参

        if(P[i][j] && A[i][j]) ans.push_back({i,j}); 

        //上下左右深搜
        if(i-1 >= 0 && M[i-1][j] >= M[i][j]) dfs(M, visited, i-1, j);
        if(i+1 < n && M[i+1][j] >= M[i][j]) dfs(M, visited, i+1, j); 
        if(j-1 >= 0 && M[i][j-1] >= M[i][j]) dfs(M, visited, i, j-1);
        if(j+1 < m && M[i][j+1] >= M[i][j]) dfs(M, visited, i, j+1); 
    }
};
```

#### bfs次短路

leetcode 5905
```
城市用一个 双向连通 图表示，图中有 n 个节点，从 1 到 n 编号（包含 1 和 n）。图中的边用一个二维整数数组 edges 表示，其中每个 edges[i] = [ui, vi]表示一条节点ui 和节点vi 之间的双向连通边。每组节点对由 最多一条 边连通，顶点不存在连接到自身的边。穿过任意一条边的时间是 time分钟。

每个节点都有一个交通信号灯，每 change 分钟改变一次，从绿色变成红色，再由红色变成绿色，循环往复。所有信号灯都同时 改变。你可以在 任何时候 进入某个节点，但是 只能 在节点信号灯是绿色时 才能离开。如果信号灯是 绿色 ，你 不能 在节点等待，必须离开。

第二小的值 是严格大于 最小值的所有值中最小的值。

例如，[2, 3, 4] 中第二小的值是 3 ，而 [2, 2, 4] 中第二小的值是 4 。
给你 n、edges、time 和 change ，返回从节点 1 到节点 n 需要的 第二短时间 。

注意：

你可以 任意次 穿过任意顶点，包括 1 和 n 。
你可以假设在 启程时 ，所有信号灯刚刚变成 绿色 。
```

图可以看做边权相等的图，想到用Bfs求最短路或者次短路。

数据存储单位可以是pair, 分别表示该节点号和起始节点到该节点的用时。使用`make_pair`进行存储

处理红绿灯，如果`targetTime / change) % 2`为奇数，说明是红灯，需要等待一段时间

* 次短路，可以使用两个`map`，分别存储当前节点号和起始节点到该节点的用时, 分别是最短用时和次短用时。

* 最短用时和次短用时都会进入队列进行bfs, 由于bfs的特性，**到达n的最短用时一定会先出现**。所以只需要等到第二个访问n的就是次短用时。

```cpp
class Solution {
public:
/// 分别记录到达某点的最短路和次短路
    unordered_map<int, int> fast;
    unordered_map<int, int> second;
    int secondMinimum(int n, vector<vector<int>>& edges, int time, int change) {
        queue<pair<int, int>> Q; // (node, time)
        /// 记录node, time
        Q.push(make_pair(1, 0));

        /// 节点邻接表
        unordered_map<int, vector<int>> G;
        int first = -1;
        
        for (auto edge: edges) {
            if (G.find(edge[0]) == G.end()) G[edge[0]] = vector<int>(0);
            if (G.find(edge[1]) == G.end()) G[edge[1]] = vector<int>(0);
            /// G是邻接表
            G[edge[0]].push_back(edge[1]);
            G[edge[1]].push_back(edge[0]);
        }
        
        while(!Q.empty()) {
            pair<int, int> p = Q.front();
            Q.pop();
            int node = p.first;
            int curTime = p.second;
            /// 可以继续走的节点
            for (auto next: G[node]) {
                /// 边权相等,BFS先访问到n到达的一定是最短路, 第二次访问到的为次短路, 依次类推
                if (next == n) {
                    if (first == -1) {
                        first = curTime + time;
                    } else {
                        if (curTime + time > first) return curTime + time;
                    }
                }
                int targetTime = curTime + time;
                /// 等待红灯
                if ((targetTime / change) % 2 == 1) {
                    /// 向上取整乘change
                    targetTime = (targetTime / change + 1) * change;
                }
                /// 设置最短路, 并入队
                if (fast.find(next) == fast.end()) {
                    fast[next] = targetTime;
                    Q.push(make_pair(next, targetTime));                
                    continue;
                }
                /// 设置次短路
                if (second.find(next) == second.end() && fast[next] < targetTime) {
                    second[next] = targetTime;
                    Q.push(make_pair(next, targetTime));                
                    continue;
                }
            }
        }
        
        return -1;
    }
};
```

可以使用优先队列(小顶堆)记录最短距离。到每个点的用时实际上只与上一个点有关，图的问题基本是这样的。

BFS, DFS重要的是剪枝，BFS入队如果没有剪枝，队的大小越来越大，不可能出现`Q.empty()`，从而发生死循环。本题的剪枝起始就是选取适当的元素入队，换言之，最短距离和次短距离入队`if(dist[to]>len+1)`,`else if(dist[to]<len+1 && dist2[to]>len+1)` ，这两个条件使队列不会无限膨胀

* `priority_queue`保证每次队中距离最小的点出队, 本身基于bfs的出队顺序也是距离从小到大的。**但优先队列可以使边权不一致时按照距离从小到达出队**

```cpp
class Solution {
public:
    typedef pair<int,int>PII;
    int secondMinimum(int n, vector<vector<int>>& edges, int time, int change) {
        // 1. 建图, 邻接表
        vector<int>G[10002];
        for(auto&e:edges){
            G[e[0]].push_back(e[1]);
            G[e[1]].push_back(e[0]);
        }
        
        // 2. 求最短路和次短路
        priority_queue<PII, vector<PII>, greater<PII>>Q;   /// 默认大顶堆
        int dist[10002],dist2[10002];   /// 储存最短路和次短路

        for(int i=1;i<=n;i++)dist[i]=dist2[i]=1e9;
        dist[1]=0;
        Q.push(PII(-dist[1], 1));
        while(!Q.empty()){
            int u=Q.top().second;
            int len=Q.top().first;
            Q.pop();    /// 统计过的出队，
            if(dist2[u]<len)continue;//比次短路还长的路，直接扔了

            for(auto&to:G[u]){
                if(dist[to]>len+1){
                    dist[to]=len+1; // 路径长度+1
                    Q.push(PII(dist[to],to));
                }else if(dist[to]<len+1 && dist2[to]>len+1){
                    dist2[to]=len+1;
                    Q.push(PII(dist2[to],to));
                }
            }
        }
        int second=dist2[n];//次短路
        
        // 3. 计算走second步的耗时
        int cost=0;
        for(int i=1;i<=second;i++){
            if((cost/change)&1)
                cost=(cost/change+1)*change;//遇到红灯，直接跳到绿灯起始时间。
            cost+=time;
        }
        return cost;
    }
};
```

#### 区间dp

区间DP，顾名思义是在区间上DP，它的主要思想就是先在小区间进行DP得到最优解，然后再利用小区间的最优解合并求大区间的最优解。

* 石子合并问题

```
有N堆石子排成一排，每堆石子有一定的数量。现要将N堆石子并成为一堆。合并的过程只能每次将相邻的两堆石子堆成一堆，每次合并花费的代价为这两堆石子的和，经过N-1次合并后成为一堆。求出总的代价最小值。
```

基本的思路是大合并区间能从小合并区间组成, 定义状态`dp[i][j]`为从第i个石子到第j个石子的合并最小代价。那么`dp[i][j] = min(dp[i][k] + dp[k+1][j])+sum[i..j]`。例如, `dp[1,2]=dp[1,1]+dp[2,2]+sum[1,2];`, `dp[1,3]=dp[1,2]+dp[3,3]+sum[1,3]或dp[1,3]=dp[1,1]+dp[2,3]+sum[1,3]`取其最优`

需要枚举起始点，长度，分隔位置，三层for循环

```cpp
int main ()
{
    int dp[210][210],sum[210][210],a[210];
    int n;
    while(~scanf("%d",&n))
    {
        //memset(dp,MAX,sizeof(dp));
        for (int i=1; i<=n; i++)
            scanf("%d",&a[i]);
        for (int i=1; i<=n; i++)
        {
            dp[i][i]=0;//初始化为0
            sum[i][i]=a[i];//将每堆石子的个数赋值进来
        }
        for (int len=1; len<n; len++)//按长度从小到大枚举
        {
            for (int i=1; i<=n&&i+len<=n; i++)//i表示开始位置
            {
                int j=len+i;                    //j表示长度为len的一段区间的结束位置
                for (int k=i; k<j; k++)         //用k来表示分割区间
                {
                    sum[i][j]=sum[i][k]+sum[k+1][j];
                    if (dp[i][j]<dp[i][k]+dp[k+1][j]+sum[i][j]) // dp[i][j]的最小分隔位置
                        dp[i][j]=dp[i][k]+dp[k+1][j]+sum[i][j];
                    //cout<<i<<" "<<j<<" "<<sum[i][j]<<" "<<k<<" "<<dp[i][j]<<endl;
                }
            }
        }
        cout<<dp[1][n]<<endl;
    }
    return 0;
}
```

#### 状压dp

* 用二进制来枚举所有可能的情况, 二进制作为mask状态

```cpp
求把 N×M 的棋盘分割成若干个 1×2 的的长方形，有多少种方案。

例如当 N=2，M=4 时，共有 5 种方案。当 N=2，M=3 时，共有 3 种方案。
```

找到所有横放的 1x2 小方格的方案数(且能保证可以竖放完全)，那么竖放的方案就是唯一的。

`dp[i][j]`：枚举第 i 列的j 状态,用长度为N的二进制数表示，状态的个数为2^n。 j 状态为第 i-1 列有 1x2 的小方格伸到了第 i 列中的所在行。每一行对应一个二进制位

当i列的状态被记 k 为第 i-1 列的 j，当 k = 10010，代表第 i-2 列中第一行、第四行有 `1x2` 的小方格捅到了第 i-1 列。同理, j表示第i-1列捅到了i列。这影响到第i-1列的状态, 也就是说第i-1列的状态受到dp[i]和dp[i-1]的同时影响。第 i 列的 二进制 与第 i-1 列的 二进制 在相同位上不能同时为 1, 不然会重叠。即`(j&k)==0`

确保能完全放置2*1的竖块, 本列的白块二进制数可以表示为`j|k`(要么是上一列捅到了本列，要么是本列开始), 其不能存在连续奇数个 0，这样才能保证竖放小格子能成功。判断条件为 j|k 不存在连续奇数个 0。

* 初始化`f[0][0] = 1;`第0列不可能存在`i-1`列深入到第i列,只有可能它深入别人
* 输出`f[m][0]`,表示第m-1列不能有伸出来的, 第m+1列状态只能全部为0
* 状态由前一种状态转化而来

```cpp
const int N = 12, M = 1 << N;

int n, m;
long long f[N][M];
bool st[M];

int main() {
    while (cin >> n >> m, n || m) {
        memset(f, 0, sizeof f);
        
        /// 所有状态1<<n个, 某个二进制如果有如果奇数个0返回false,方便st[j | k]使用
        for (int i = 0; i < 1 << n; ++i) {      
            st[i] = true;
            int cnt = 0;                        
            for (int j = 0; j < n; ++j) {
                if (i >> j & 1) {               
                    if (cnt & 1) st[i] = false; 
                    cnt = 0;                    
                }                   
                else cnt ++;
                
            }
            if (cnt & 1) st[i] = false;
        }
        
        f[0][0] = 1;    /// f[0][1..]等其他状态均为0，因为不存在
        for (int i = 1; i <= m; ++i) /// 遍历列
            for (int j = 0; j < 1 << n; ++j)    /// j,k分别表示第i列和第i-1列的状态，大小均为2^n
                for (int k = 0; k < 1 << n; ++k)
                    if ((j & k) == 0 && st[j | k]) /// 如果j&k==0则可以放置1*2, 如果st[j|k]则可以放置2*1, 也就是说这样j,k相邻列的状态都符合要求, 换句话说这是检验j,k这两种状态在i-1和i列上是否合法
                        f[i][j] += f[i - 1][k]; ///
        
        cout << f[m][0] << endl;
    }
    return 0;
}
```

#### 树形dp

1. 树形dp小范围内子节点和根节点之间有状态转移关系

2. 树形dp一般每个边会有权值, 思想是 子树节点往往将信息传递给根节点, 树从叶子节点完成到根节点的状态转移将得到最优权值。

没有上司的舞会
```
某大学有 N 名职员，编号为 1~N。
他们的关系就像一棵以校长为根的树，父节点就是子节点的直接上司。
每个职员有一个快乐指数H。
现在要召开一场周年庆宴会，不过，没有职员愿意和直接上司一起参会。
在满足这个条件的前提下，主办方希望邀请一部分职员参会，使得所有参会职员的快乐指数总和最大，求这个最大值。

输入格式
第一行一个整数N
接下来N行表示编号i的快乐指数
接下来N-1行，每行两个数L,R, 表示R为L的直接上司
```

* 定义`f[u][0/1]`表示从u的子结点中选，并且选u或者不选u(0表示不选，1表示选)的方案权值的最大值。
* 不选u, 则可以选择子节点(也可以不选), 即`f[u][0] = max(f[子节点][0], f[子节点][1])`
* 选择u, 则不能选子节点`f[u][1] = w[u] + f[子节点][0]`

* 直接用图的邻接表储存树结构就可以

```cpp
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e5+5;
const int INF = 0x3f3f3f3f;
int n, w[MAXN], dp[MAXN][2];
vector<int> G[MAXN];
void dfs(int u, int fa)
{
    dp[u][0] = 0;
    dp[u][1] = w[u];
    for (auto v: G[u])
    {
        if (v != fa)    /// 注意访问时不要访问到父节点访问儿子节点
        {
            dfs(v, u);      /// 一直执行到每个叶子节点，然后自底而上
            dp[u][0] += max(dp[v][0], dp[v][1]);
            dp[u][1] += dp[v][0];
        }
    }
}
int main()
{
    scanf("%d", &n);
    for (int i = 1; i <= n; i++) scanf("%d", &w[i]);
    int u, v;
    while (~scanf("%d%d", &u, &v) && (u+v))
    {
        G[u].push_back(v);
        G[v].push_back(u);
    }
    dfs(1, -1); /// 从节点1也就是根节点开始
    int ans = max(dp[1][0], dp[1][1]);  /// 选择，或者不选
    printf("%d\n", ans);
    return 0;
}
```

```
一番侦察之后，聪明的小偷意识到"这个地方的所有房屋的排列类似于一棵二叉树"。 如果两个直接相连的房子在同一天晚上被打劫，房屋将自动报警。

计算在不触动警报的情况下，小偷一晚能够盗取的最高金额。

输入: [3,2,3,null,3,null,1]

     3
    / \
   2   3
    \   \ 
     3   1

输出: 7 
解释:小偷一晚能够盗取的最高金额 = 3 + 3 + 1 = 7.
```

典型的树形dp, 每个节点有权值, 子树下根节点值(最高金额)取决于根节点和子节点。

根节点选择, 则子节点不能选择; 根节点不选择, 子节点可以选, 也可以不选。

注意leetcode不用建树, 但是写程序时应该考虑树的情况。一般将树建成图那种邻接表就可以，访问时注意不要访问到父节点。

注意树形dp一般是自底向上的, 也就是计算时在计算得到左子节点右子节点的数值后, 根据两者计算得到根节点数值。因此在递归时, 我们要先递归到底部(访问到叶子节点), 然后自底向下计算, 最后计算根节点。

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    unordered_map<TreeNode*, vector<int>> dp;
    int rob(TreeNode* root) {
        dfs(root);
        return max(dp[root][0], dp[root][1]); 
    }

    void dfs(TreeNode* root) {
        /// map的id可以直接赋值
        dp[root].resize(2);
        // dp[node][0]表示不访问这个节点
        // dp[node][1]表示访问这个节点
        dp[root][0] = 0;
        dp[root][1] = root->val;
        
        /// 左儿子
        if (root->left) {
            dfs(root->left);    /// 一直访问到叶子节点
            dp[root][0] += max(dp[root->left][0], dp[root->left][1]);
            dp[root][1] += dp[root->left][0];
        }
        if (root->right) {
            dfs(root->right);
            dp[root][0] += max(dp[root->right][0],dp[root->right][1]);
            dp[root][1] += dp[root->right][0];
        }
    }
};
```
