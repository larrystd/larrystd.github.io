### numpy

#### 使用

不同于python列表，numpy要求数组必须包含同一类型的数据。
```py
np.array([1, 2, 3, 4], dtype='float32')

np.zeros(10, dtype=int)	# 长为10的数组，值为0

np.ones((3,5), dtype=float) # 3*5的浮点型数组

np.linspace(0, 1, 5) # 五个元素数组，五个元素在0~1均匀分配

np.arrange(0, 20, 2) # 0开始20结束，步长为2

np.random.seed(0)  # 随机数种子

np.random.random((3, 3))  # 3*3的随机数组,0~1均匀分布

np.random.normal(0, 1, (3,3)) # 3*3, 均值0, 方差1的正态分布随机数组

np.random.randint(0, 10, (3, 3)) # 3*3 [0, 10)区间的随机数组、

np.eye(3) # 3*3单位矩阵
```

```py

[[1, 1, 1], [1, 1, 1]]
```

布尔数据操作

`np.sum(x<6)`这个语句可以计算x<6的个数，因为`true`被解释成1, `false`被解释成0。

布尔数组可以做为掩码`x[x < 5]`可以把x<5的值从数组中选出。

### pandas
pandas可以看成增强版Numpy结构化数组，行列不只是简单的整数索引，还可以带上标签。

#### Series对象
Series对象是一个带索引数据构成的一维数组,可以用一个数组创建Series对象
```py
data = pd.Series([0.25, 0.5, 0.75, 1.0])

data.values	# 返回array
data.index # 返回Index对象

data[0]
data[1:3] # 可以切片

data.keys()
data.items()

data['a':'c']
data[(data>0.3) & (data < 0.8>)]
```
Series可以看成特殊的字典

```py
data = pd.Series([0.25, 0.5, 0.75, 1.0], index = ['a', 'b', 'c', 'd'])

data['a']

data = pd.Series({'a':1, 'b':2})
```
可以直接通过dict来创建Series

#### DataFrame对象

DataFrame可用认为是二维的Series, 
```py
population = pd.Series(population_dict)
area = pd.Series(area_dict)
states = pd.DataFrame({'population' : population, 'area' : area})

states.index
state.columns # 列标签
```

创建方法, 如果有键不存在，会用NAN表示

Index对象可用看作是不可变数组，pd.index, pd.columns返回的都是Index对象，相比于Numpy数组，Index是不可变的，其他基本一致。

#### 缺失值

* None是一个python单体对象，常在代码中表示缺失值，但由于是一个python对象，不能表示Numpy/Pandas数组类型缺失值，只能用于object数组
* NaN, 任何数与NaN操作，结果均为NaN。Pandas会将没有标签值的数据自动转为nan

缺失值处理
```py
isnull() #发现缺失值
data.dropna() #剔除缺失值， 默认剔除整行
data.dropna(axis='columns') #剔除整列

fillna() #填充缺失值
filna(method='ffill', axis=1)
```
pandas除了数值方法，还有字符串方法。一般储存的类型不是数值就是字符串。

### Conda 包管理

Python的版本比较多，并且它的库也非常广泛，同时库和库之间存在很多依赖关系，所以在库的安装和版本的管理上很麻烦。Conda是一个管理版本和Python环境的工具。

#### 环境管理

* 创建环境
```sh
# 创建一个名为python34的环境，指定Python版本是3.5（不用管是3.5.x，conda会为我们自动寻找3.５.x中的最新版本）
conda create --name py35 python=3.5
```
* 激活环境
```py
# 安装好后，使用activate激活某个环境
activate py35 # for Windows
source activate py35 # for Linux & Mac
(py35) user@user-XPS-8920:~$
 # 激活后，会发现terminal输入的地方多了py35的字样，实际上，此时系统做的事情就是把默认2.7环境从PATH中去除，再把3.4对应的命令加入PATH

(py35) user@user-XPS-8920:~$ python --version
```

* 返回主环境
```py
deactivate py35 # for Windows
source deactivate py35 # for Linux & Mac
```

* 删除环境

<!-- more -->

```py
# 删除一个已有的环境
conda remove --name py35 --all
```

查看系统中的所有环境

#### 包管理

* 安装库
```py
conda install numpy
# conda会从从远程搜索numpy的相关信息和依赖项目
```

* 查看已经安装的库

```py
# 查看已经安装的packages
conda list
# 最新版的conda是从site-packages文件夹中搜索已经安装的包，可以显示出通过各种方式安装的包
```

* 查看某个环境的已安装包 `conda list -n py35`

* 搜索package的信息
`conda search numpy`

* 安装package到指定的环境
```py
# 安装package
conda install -n py35 numpy
# 如果不用-n指定环境名称，则被安装在当前活跃环境
# 也可以通过-c指定通过某个channel安装
```

* 更新package
`conda update -n py35 numpy`

* 删除package
`conda remove -n py35 numpy`

* 更新conda
` conda update conda`
