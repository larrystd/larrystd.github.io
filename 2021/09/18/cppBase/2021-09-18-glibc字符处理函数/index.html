glibc字符串

2021-09-18
cpp base
c, glibc

glibc是C语言的库函数, 持有系统调用的底层接口和常用处理函数。cpp也许不常用malloc, 但字符串函数常常在cpp使用

string

字符串实际上是使用 null 字符 \0 终止的一维字符数组, 换言之字符串中存在除ASCII码=0意外的其它255个ASCII字符

strcpy 拷贝字符串

strlen 求字符串长度

以\0为界

strncmp 安全的字符串比较

用memcmp可能更好

strpbrk 从s中找到accept中的字符, 返回第一个的地址,

当s为字符时可以充当匹配, 如strpbrk("a", "abc");

strstr和strrchr 找到最后一次出现的字符

strstr可判断字符在字符串中是否存在, A pointer to the first occurrence in str1 of the entire sequence of characters specified in str2, or a null pointer if the sequence is not present in str1.

strtok 字符串分割 将分割符传入

当strtok()在参数s 的字符串中发现到参数delim 的分割字符时则会将该字符改为\0 字符。在第一次调用时，strtok()必需给予参数s 字符串，往后的调用则将参数s 设置成NULL。

strtok执行一次将匹配分割delim位置置\0, 并返回起始指针, 这时候起始指针所表示的字符段就是分割的字符段。例如abc-=-def执行一次strtok(s,”-“),形成abc\0=-def返回起始指针s, 这时候strlen(s)为3, printf(s)为abc。

使用,将str字符以spl分割,存于dst中，并返回子字符串数量

  • C++的find可以起到寻找字符串的作用, 可以查找子串和字符.string::npos用来表示不存在的位置

字符串转型 atof, atoi sprintf

double atof(const char* str):字符串转浮点数

int atoi/atol/atoll(const char* str):字符串转整型

整型等转字符串用sprintf

Memory级别的字符

memcpy

该文件用宏定义定义了字节拷贝和字拷贝

memcpy 内存字节拷贝

memcpy的实现比较简单, 基本思路就是从src开始按字节拷贝到destination

为了加速拷贝, 这里有个块的概念先拷贝大块, 剩余字节用小块拷贝

memcpy可能导致内存覆盖的问题, 返回目的地起始地址

memmove 无覆盖的内存字节拷贝

memmove不是移动, 只是在memcpy基础上, 可以防止地址段重叠前向拷贝造成的覆盖问题。memove可以防止memcpy的拷贝, 一般如果将数据拷贝到开辟的新内存, 用memcpy; 如果还是这个内存, 例如array的插入所有字符向后移动一位, 用memove。

关键区别是, memcpy用前向拷贝BYTE_COPY_FWD; 而memmove用的是后向拷贝BYTE_COPY_BWD。当内存发生局部重叠的时候，memmove保证拷贝的结果是正确的，memcpy不保证拷贝的结果的正确

memset 内存字节赋值

使用方法memset(a, 0, sizeof(a)), 块状赋值。由于采用了内部优化, 初始化为0或者-1, 不能初始化成别的数。
