Cpython垃圾回收主要算法是引用计数，每个对象都会统计多少引用指向自己，当引用计数归零对象会被立即销毁。Cpython调用对象的`__del__`方法，释放分配给对象的内存。Cpython2增加了分代垃圾回收算法。

有时候需要引用对象而不让对象存在时间超过所需要时间，这时候用弱引用，类似于cpp的`weak_ptr`。可以解决`shared_ptr`循环引用的问题，如果需要调用将`weak_ptr`提升为`shared_ptr`，不用了就reset `shared_ptr`，也可以避免对象一只被`shared_ptr`而无法销毁的问题。



### 装饰器

@ 符号是装饰器的语法糖

装饰器是可调用的对象，其参数是被装饰的函数。以下inner()是deco()的闭包

```py
def deco(func):
    print('running deco(%s)' % func)
    print ('abc')
    def inner():
        print ('running inner()')
    return inner

@deco       # 装饰器相当于先调用函数deco(target), 然后用deco返回的闭包代替当前target函数, 注意deco必须返回一个闭包对象, 不然报错
def target():
    print ('running target()')

target()

# 输出
running register(<function target at 0x7fd393c55d08>)
abc
running inner()
```
以上使用装饰器, `@deco`相当于调用了`target = deco(target)`, 相当于提高了deco函数可重用性。而target函数实际被inner()函数取代了, 因此加了`@deco`之后, def target等价为deco(target) + inner(), 原函数的信息已经没有了

装饰器一个关键特性是，它在被装饰的函数定义之后立即执行。
这时候执行不执行函数，因为函数还未定义好。这里有个`导入`和`运行`时的区别。

带参数的装饰器

```py
		num = func(*args,**kwargs)+1 #调用实参函数即test(1,2)+1，并传入一致的实参，并且用变量来接收原函数的返回值，
		return num #将接受到的返回值再次返回到新的test()函数中。
```

* 一个简单的装饰器，输出函数运行时间

```py
# clockdeco.py
import time

def clock(func):
    def clocked(*args):
        t0 = time.perf_counter()
        result = func(*args)
        elapsed = time.perf_counter() - t0
        name = func.__name__
        arg_str = '. '.join(repr(arg) for arg in args)
        print('[%0.8fs] %s(%s) -> %r' % (elapsed, name, arg_str, result))
        return result       # result会置入调用闭包的函数, 也就是snooze(result)中
    return clocked

# demo.py
import time

@clock
def snooze(seconds):  # 想到于转而调用clocked(seconds)

def factorial(n):
```

#### setter和getter

Python中不存在只能从对象内部访问的“私有(private)”变量，但Python制定了以下公约：

以单下划线开头的成员变量或方法，是设计者不想暴露给外部的API，使用者不应该进行访问。但类实例和子类实例能访问到这些变量

以双下划线开头且以至多一个下划线结尾的成员变量或方法，会进行名称转写。可以通过名称改写访问到

python中的私有变量和私有方法仍然是可以访问的；访问方法（名称改写）如下： 私有变量:实例._类名__变量名; 私有方法:实例._类名__方法名()

### 可迭代 和生成器
所有生成器都是迭代器，因为生成器完全实现了迭代器的接口。迭代器用来从集合中取元素，生成器则用于生成元素。

```py
import re
import reprlib

RE_WORD = re.compile('\w+')

class Sentence:
    def __init__(self, text):
        self.text = text
        self.words = RE_WORD.findall(text)
    def __getitem__(self, index)
        return self.words[index]
    def __len__(self):
        return len(self.words)
    def __repr__(self):
        return 'Sentence(%s)' % reprlib.repr(self.text)

s = Sentence('The time has come')
for word in s:
    print(word)

list(s)
```
`re.finall`函数返回正则表达式全部非重叠匹配后的字符串列表。self.words为字符串列表结果。

解释器需要迭代对象x时，会自动调用`iter(x)`。如果对象实现了`__iter__`就调用之，若没有实现但实现了`__getitem__`方法，python会创建一个迭代器按顺序获取元素。

* 标准的迭代器接口有两个办法

`__next__` 返回下一个可用的元素，如果没有元素，抛出StopIteration异常

`__iter__` 返回self，以便在该应用迭代器地方使用迭代器。

```py
class SentenceIterator:
    def __init__(self, words):
        self.words = words
        self.index = 0
    def __next__(self):
        try:
            word = self.words[self.index]
        except IndexError:
            raise StopIteration()
        self.index += 1
```

#### 生成器到协程

Cpython解释器本身不是线程安全的，因此有GIL: 全局解释器锁（英语：Global Interpreter Lock，缩写GIL），它使得任何时刻仅有一个线程在执行。[1]即便在多核心处理器上，使用 GIL 的解释器也只允许同一时间执行一个线程。由于GIL的存在，导致Python多线程性能甚至比单线程更糟。
但python线程等待操作系统返回结果时会释放GIL，例如等待网络响应，这时其他线程可以执行，意味着python多线程能获得些许提升。

于是出现了协程（Coroutine）。协程的作用，是在执行函数A时，可以随时中断，去执行函数B，然后中断继续执行函数A（可以自由切换）。但这一过程并不是函数调用（没有调用语句），这一整个过程看似像多线程，然而协程只有一个线程执行.

如上，方法中包含yield表达式后，Python会将其视作generator对象，不再是普通的方法。生成器这种执行一次就阻塞等待`__next__`的特性可用作协程。

```py
def test():
    print("generator start")
    n = 1
    while True:
        yield_expression_value = yield n
        print("yield_expression_value = %d" % yield_expression_value)
        n += 1


# ①创建generator对象
generator = test()
print(type(generator))

print("\n---------------\n")

# ②启动generator
next_result = generator.__next__()
print("next_result = %d" % next_result)

print("\n---------------\n")

# ③发送值给yield表达式
send_result = generator.send(666)
print("send_result = %d" % send_result)

<class 'generator'>
---------------
generator start
next_result = 1
---------------
yield_expression_value = 666
send_result = 2
```
启动generator后，代码将执行到yield出现的位置，也就是执行到yield n，然后将n传递到generator.__next__()这行的返回值。生成器执行到yield n后将暂停在这里，直到下一次生成器被启动。

调用send方法可以发送值给yield表达式，同时恢复生成器的执行。生成器从上次中断的位置继续向下执行，然后遇到下一个yield，生成器再次暂停，切换到主函数打印出send_result。

生产者，消费者模型。模式是生产一个，消费一个，这不会阻塞，而是线程自动执行一次producer，再执行一次comsumer,循环往复。

```py
def consumer():
    print("[CONSUMER] start")
    r = 'start'
    while True:
        n = yield r
        if not n:
            print("n is empty")
            continue
        print("[CONSUMER] Consumer is consuming %s" % n)
        r = "200 ok"


def producer(c):
    # 启动generator
    start_value = c.send(None)
    print(start_value)
    n = 0
    while n < 3:
        n += 1
        print("[PRODUCER] Producer is producing %d" % n)
        # 唤醒yield
        r = c.send(n)
        print('[PRODUCER] Consumer return: %s' % r)
    # 关闭generator
    c.close()


# 创建生成器
c = consumer()
# 传入generator
producer(c)
```
