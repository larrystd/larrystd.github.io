### 博弈dp

#### leetcode 486 预测赢家

```
给你一个整数数组 nums 。玩家 1 和玩家 2 基于这个数组设计了一个游戏。

玩家 1 和玩家 2 轮流进行自己的回合，玩家 1 先手。开始时，两个玩家的初始分值都是 0 。每一回合，玩家从数组的任意一端取一个数字(即，nums[0] 或 nums[nums.length - 1])，取到的数字将会从数组中移除（数组长度减 1 ）。玩家选中的数字将会加到他的得分上。当数组中没有剩余数字可取时，游戏结束。

如果玩家 1 能成为赢家，返回 true 。如果两个玩家得分相等，同样认为玩家 1 是游戏的赢家，也返回 true 。你可以假设每个玩家的玩法都会使他的分数最大化。

1 <= nums.length <= 20
0 <= nums[i] <= 10^7
```

这是个零和博弈, 无论采取何种策略，博弈双方的总得分不变。即一个人得分变多，另一个必然减少。零和博弈问题的关键都是要找出如何让后手更差的局面的方法。让后手更差其实就是让先手更优。每次轮次都为当前选手的最优策略, 以使对手得分最少。

零和博弈自己得分多和对手得分少同时存在的，因为总分一致。可以设置`dp[L][R]`为在`[L，R]`段的最佳选择，也就是能得到最高的分。而选择之后对手为`dp[L+1][R]`，要么是`dp[L][R-1]`。因此此步目标是`min(dp[L+1][R], dp[L][R-1])`使对手得分最小, 此时先手得分为`sum(L, R)-min(dp[L+1][R], dp[L][R-1])`

可以看出子串`[L,R]`可能被重复计算多次，因此可用动态规划求解。

<!-- more -->

```cpp
class Solution {
public:
    vector<vector<int>> memo;
    bool PredictTheWinner(vector<int>& nums) {
        int n = nums.size();
        memo.assign(n, vector<int>(n, -1));

        return 2 * dfs(0, n-1, nums) >= sumNum(0, n-1, nums);
    }

    // dfs(l,r)表示在[l,r]区间中得分高的那一个
    int dfs(int l, int r, const vector<int>& nums) {
        if (memo[l][r] != -1)
            return memo[l][r];
        
        if (l == r)
            return nums[l];
        return memo[l][r] = sumNum(l, r, nums) - min(dfs(l+1, r, nums), dfs(l, r-1, nums)); // 让自己的得分最高, 就是选择对面得分较少的那一个

    }

    int sumNum(int l, int r, const vector<int>& nums) {
        int ret = 0;
        for (int i = l; i <= r; i++) {
            ret += nums[i];
        }
        return ret;
    }
};
```

#### leetcode464 我能赢吗

```
在 "100 game" 这个游戏中，两名玩家轮流选择从 1 到 10 的任意整数，累计整数和，先使得累计整数和 达到或超过100 的玩家，即为胜者。

如果我们将游戏规则改为"玩家 不能 重复使用整数"呢？

例如，两个玩家可以轮流从公共整数池中抽取从 1 到 15 的整数（不放回），直到累计整数和 >= 100。

给定两个整数maxChoosableInteger（整数池中可选择的最大数）和desiredTotal（累计和），若先出手的玩家是否能稳赢则返回 true，否则返回 false 。假设两位玩家游戏时都表现 最佳 。

输入：maxChoosableInteger = 10, desiredTotal = 11
输出：false
```

同样类似上题的博弈, 只是选择的数字不能重复。

特判：

* 如果最大能选择的数字maxChoosableInteger比期望的总数desiredTotal要大，先手稳赢，返回True
* 如果能选择的所有数字总和比期望的总数desiredTotal要小，一定到达不了desiredTotal，先手稳输，返回False(因为不能选择重复数字)

状态, 对于n个数字一共有2^n个状态, 因此记忆化数组需要开辟2^n的空间, dp[state]表示在state选择下当前先手能否赢。

```py
class Solution:
    def canWin(self, maxChoosableInteger:int, desiredTotal: int) -> bool :
        if maxChoosableInteger >= desiredTotal:
            return True 
        if sum(range(1,maxChoosableInteger+1)) < desiredTotal:
            return False
        
        def dfs(state, desiredTotal, dp):
            if dp[state] != None:
                return dp[state]
            
            for i in range(1, maxChoosableInteger+1):
                cur = 1 << (i-1)
                if cur & state != 0:    # 已经选过了
                    continue
                if i >= desiredTotal or not dfs(state | cur, desiredTotal-i, dp):   # 对手输, i>=desiredTotal这里很重要, 相当于自底而上的底, 返回True
                    dp[state] = True 
                    return True 
            dp[state] = False # 在当前state下先手只有输
            return False

        return dfs(0, desiredTotal, [None] * (1 << maxChoosableInteger))
        
```

### 区间dp

区间DP，顾名思义是在区间上DP，它的主要思想就是先在小区间进行DP得到最优解，然后再利用小区间的最优解合并求大区间的最优解。区间dp和归并排序类似, 初始条件都是单个元素组成的区间, 自底向上。

#### 石子合并问题

```
有N堆石子排成一排，每堆石子有一定的数量。现要将N堆石子并成为一堆。合并的过程只能每次将相邻的两堆石子堆成一堆，每次合并花费的代价为这两堆石子的和，经过N-1次合并后成为一堆。求出总的代价最小值。
```

基本的思路是大合并区间能从小合并区间组成, 定义状态`dp[i][j]`为从第i个石子到第j个石子的合并最小代价。那么`dp[i][j] = min(dp[i][k] + dp[k+1][j])+sum[i..j]`。例如, `dp[1,2]=dp[1,1]+dp[2,2]+sum[1,2];`, `dp[1,3]=dp[1,2]+dp[3,3]+sum[1,3]或dp[1,3]=dp[1,1]+dp[2,3]+sum[1,3]`取其最优`

需要枚举起始点，长度，分隔位置，三层for循环。一般是枚举区间，把区间分成左右两部分，然后求出左右区间再合并。

```cpp
// 动态规划复杂了就发现, 递归没有迭代好写了
int main ()
{
    int dp[210][210],sum[210][210],a[210];
    int n;
    while(~scanf("%d",&n))
    {
        //memset(dp,MAX,sizeof(dp));
        for (int i=1; i<=n; i++)
            scanf("%d",&a[i]);  // 每堆石子的数量
        
        // 区间dp和归并排序类似, 因此初始就是单个石子堆了, 自底向下
        for (int i=1; i<=n; i++)
        {
            dp[i][i]=0;//初始化为0
            sum[i][i]=a[i];//将每堆石子的个数赋值进来
        }
        for (int len=1; len<n; len++) // 按长度从小到大枚举
        {
            for (int i=1; i<=n&&i+len<=n; i++)//i表示开始位置, [i,i+len]表示一段区间(一共n堆石子)
            {
                int j=len+i;                    //j表示长度为len的一段区间的结束位置
                for (int k=i; k<j; k++)         //用k来表示分割区间
                {
                    sum[i][j]=sum[i][k]+sum[k+1][j];    // [i,j]这个集合的堆石子的花费
                    if (dp[i][j]<dp[i][k]+dp[k+1][j]+sum[i][j]) // dp[i][j]的最小分隔位置
                        dp[i][j]=dp[i][k]+dp[k+1][j]+sum[i][j];
                }
            }
        }
        cout<<dp[1][n]<<endl;
    }
    return 0;
}
```

### 状压dp

用二进制来枚举所有可能的情况, 二进制mask表示状态, dp[]的索引的元素其实就是一个状态。所谓的压缩，其实就是将一个集合压缩成了一个整数的意思，因为整数可以作为数组的下标，这样操作会方便我们的编码。

#### 蒙德里安的梦想

```
求把 N×M 的棋盘分割成若干个 1×2 的的长方形，有多少种方案。

例如当 N=2，M=4 时，共有 5 种方案。当 N=2，M=3 时，共有 3 种方案。
```

我们可以只枚举1 × 2 1的骨牌横着放的方式，只要横着放的骨牌都确定了，剩余的格子就竖着放骨牌，这时候竖着放的骨牌的摆放法是唯一的。所以一个合法方案里，横着放的骨牌的摆放方法就决定了整个方案。


枚举所有符合条件的状态, 并判断是否可以选择。只考虑使用`1*2`的方格填充`dp[i][j]`表示第 i 列的j 状态, 状态用长度为N的二进制数表示，个数为2^n。 注意**j 状态为1的位置表示i 列有 1x2 的小方格伸到了第 i+1 列**。

除了j以外, 枚举一种状态k, k 为第 i-1 列的 j(即前一个列的状态)。当 k 状态为1代表第 i-1 列有小方格伸到了第 i 列。而j表示第i列捅到了i+1列。显然第 i 列的 二进制 与第 i-1 列的 二进制 在相同位上不能同时为 1, 不然会重叠。即`(j&k)==0`

确保能完全放置2*1的竖块, 本列的白块二进制数可以表示为`j|k`(要么是上一列捅到了本列，要么是本列开始), **其间不能存在连续奇数个 0，这样才能保证竖放小格子能成功。判断条件为 j|k 不存在连续奇数个 0**。 

初始化`f[0][0] = 1;`第0列不可能存在`i-1`列深入到第i列,只有可能它深入别人。**输出`f[m][0]`,表示第m-1列不能有伸出来的, 第m+1列状态只能全部为0**

状态由前一种状态转化而来, 状压dp以状态作为衡量要求的单位。

```cpp
const int N = 12, M = 1 << N;

int n, m;
long long f[N][M];
bool st[M];

int main() {
    while (cin >> n >> m, n || m) {
        memset(f, 0, sizeof f);
        
        /// 所有状态1<<n个, 某个二进制如果有如果奇数个0返回false,方便st[j | k]使用
        // 保存连续0为偶数的合法状态
        for (int i = 0; i < 1 << n; ++i) { // 遍历1<<n个状态
            st[i] = true;
            int cnt = 0;                        
            for (int j = 0; j < n; ++j) {
                if (i >> j & 1) {               
                    if (cnt & 1) st[i] = false;    // 连续0的个数为1  
                    cnt = 0;                    
                }                   
                else cnt ++;    // cnt表示连续0的个数
                
            }
            if (cnt & 1) st[i] = false;
        }
        
        f[0][0] = 1;    /// f[0][1..]等其他状态均为0，因为不存在
        for (int i = 1; i <= m; ++i) /// 遍历列, 其实是遍历了m+1列, 0~m
            for (int j = 0; j < 1 << n; ++j)    /// j,k分别表示第i列和第i-1列的状态，大小均为2^n
                for (int k = 0; k < 1 << n; ++k)    // 这些状态哪些符合要求
                    if ((j & k) == 0 && st[j | k]) /// j&k==0说明没有重叠, 如果st[j|k]则表示防止完1*2的砖块之后剩下的可以顺利放置2*1填充
                        f[i][j] += f[i - 1][k];
        
        cout << f[m][0] << endl;    // 第m的状态必须为0因为不能伸出来
    }
    return 0;
}
```

### 树形dp

1. 树形dp小范围内子节点和根节点之间有状态转移关系

2. **树形dp一般每个边会有权值, 思想是 子树节点往往将信息传递给根节点**, 树从叶子节点完成到根节点的状态转移将得到最优权值。**一般需要先遍历到叶子节点再计算, 也就是先一直dfs, 之后才是计算**。

#### 没有上司的舞会

```
某大学有 N 名职员，编号为 1~N。
他们的关系就像一棵以校长为根的树，父节点就是子节点的直接上司。
每个职员有一个快乐指数H。
现在要召开一场周年庆宴会，不过，没有职员愿意和直接上司一起参会。
在满足这个条件的前提下，主办方希望邀请一部分职员参会，使得所有参会职员的快乐指数总和最大，求这个最大值。

输入格式
第一行一个整数N
接下来N行表示编号i的快乐指数
接下来N-1行，每行两个数L,R, 表示R为L的直接上司
```

定义`f[u][0/1]`表示从u的子结点中选，并且选u或者不选u(0表示不选，1表示选)的方案权值的最大值。

1. 不选u, 则可以选择子节点(也可以不选), 即`f[u][0] = max(f[子节点][0], f[子节点][1])`
2. 选择u, 则不能选子节点`f[u][1] = w[u] + f[子节点][0]`

直接用图的邻接表储存树结构就可以, 树形dp保证顾及了树的任何一段小范围, 包括没有根节点的情况, 包括树形子序列和子串。

```cpp
#include <bits/stdc++.h>
using namespace std;
const int MAXN = 1e5+5;
const int INF = 0x3f3f3f3f;
int n, w[MAXN], dp[MAXN][2];
vector<int> G[MAXN];
void dfs(int u, int fa)  // 参数为当前节点u和父亲节点fa
{
    dp[u][0] = 0;
    dp[u][1] = w[u];    // 选择可以获得的快乐指数
    for (auto v: G[u])
    {
        if (v != fa)    /// 注意访问时不要访问到父节点访问儿子节点
        {
            dfs(v, u);      /// 一直执行到每个叶子节点，然后自底而上, 学习
            dp[u][0] += max(dp[v][0], dp[v][1]);    // 填充dp
            dp[u][1] += dp[v][0];
        }
    }
}
int main()
{
    scanf("%d", &n);
    for (int i = 1; i <= n; i++) scanf("%d", &w[i]);
    int u, v;
    // 邻接表存储
    while (~scanf("%d%d", &u, &v) && (u+v))
    {
        G[u].push_back(v);
        G[v].push_back(u);
    }
    dfs(1, -1); /// 从节点1也就是根节点开始
    int ans = max(dp[1][0], dp[1][1]);  /// 选择，或者不选根节点的最大值
    printf("%d\n", ans);
    return 0;
}
```

#### leetcode337 打家劫舍III

```
一番侦察之后，聪明的小偷意识到"这个地方的所有房屋的排列类似于一棵二叉树"。 如果两个直接相连的房子在同一天晚上被打劫，房屋将自动报警。

计算在不触动警报的情况下，小偷一晚能够盗取的最高金额。

输入: [3,2,3,null,3,null,1]

     3
    / \
   2   3
    \   \ 
     3   1

输出: 7 
解释:小偷一晚能够盗取的最高金额 = 3 + 3 + 1 = 7.
```

典型的树形dp, 每个节点有权值, 子树下根节点值(最高金额)取决于根节点和子节点。

根节点选择, 则子节点不能选择; 根节点不选择, 子节点可以选, 也可以不选。

注意leetcode不用建树, 但是写程序时应该考虑树的情况。一般将树建成图那种邻接表就可以，访问时注意不要访问到父节点。

注意树形dp一般是自底向上的, 也就是计算时在计算得到左子节点右子节点的数值后, 根据两者计算得到根节点数值。因此在递归时, 我们要先递归到底部(访问到叶子节点), 然后自底向下计算, 最后计算根节点。

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    unordered_map<TreeNode*, vector<int>> dp;
    int rob(TreeNode* root) {
        dfs(root);
        return max(dp[root][0], dp[root][1]); // 根节点选没选
    }

    void dfs(TreeNode* root) {
        /// map的id可以直接赋值
        dp[root].resize(2);
        // dp[node][0]表示不访问这个节点
        // dp[node][1]表示访问这个节点
        dp[root][0] = 0;
        dp[root][1] = root->val;
        
        /// 左儿子
        if (root->left) {
            dfs(root->left);    /// 一直访问到叶子节点
            dp[root][0] += max(dp[root->left][0], dp[root->left][1]);
            dp[root][1] += dp[root->left][0];
        }
        if (root->right) {
            dfs(root->right);
            dp[root][0] += max(dp[root->right][0],dp[root->right][1]);
            dp[root][1] += dp[root->right][0];
        }
    }
};
```

### 股票dp

#### leetcode 122 买卖股票的最佳时机 II

```
给定一个数组 prices ，其中prices[i] 是一支给定股票第 i 天的价格。

设计一个算法来计算你所能获取的最大利润。你可以尽可能地完成更多的交易（多次买卖一支股票）。

注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

输入: prices = [7,1,5,3,6,4]
输出: 7
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4。
    随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3。
```

每天交易结束后只可能存在手里有一支股票或者没有股票的状态。定义状态 `dp[i][0]` 表示第 i 天交易完后手里没有股票的最大利润，`dp[i][1]` 表示第 i 天交易完后手里持有一支股票的最大利润。

考虑 `dp[i][0]` 的转移方程，如果这一天交易完后手里没有股票，那么可能的转移状态为前一天已经没有股票，即 `dp[i-1][0]`，或者前一天结束的时候手里持有一支股票，即 `dp[i-1][1]`，这时候我们要将其卖出，并获得 `prices[i]` 的收益。因此为了收益最大化, `dp[i][0]=max{dp[i−1][0],dp[i−1][1]+prices[i]}`

再考虑 `dp[i][1]`，按照同样的方式考虑转移状态，那么可能的转移状态为前一天已经持有一支股票，即 `dp[i−1][1]`，或者前一天结束时还没有股票，即 `dp[i−1][0]`，这时候我们要将其买入，并减少 `prices[i]` 的收益。可以列出如下的转移方程
`dp[i][1]=max{dp[i−1][1],dp[i−1][0]−prices[i]}`

初始状态，可以知道第 0 天交易结束的时候 `dp[0][0]=0`，`dp[0][1]=−prices[0]`。股票问题类似01背包问题, 都会对买或者不买做出决策。

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int n = prices.size();
        int dp[n][2];
        dp[0][0] = 0, dp[0][1] = -prices[0];

        for (int i = 1; i < n; ++i) {
            dp[i][0] = max(dp[i - 1][0], dp[i - 1][1] + prices[i]);
            dp[i][1] = max(dp[i - 1][1], dp[i - 1][0] - prices[i]);
        }
        return dp[n - 1][0];
    }
};
```

* 贪心的方法

整个问题等价于寻找 x 个不相交的区间 `(l_i,r_i]` 使得区间右值减左值之和最大化(即`l_i`天买入,`r_i`天卖出); **每次选择大于0的小区间, 就可以实现不相交区间差的最大化**, 即长区间可以看成是大于0的区间长度为1的小区间累积而成。

```cpp

class Solution {
public:
    int maxProfit(vector<int>& prices) {   
        int ans = 0;
        int n = prices.size();
        for (int i = 1; i < n; ++i) {
            ans += max(0, prices[i] - prices[i - 1]);
        }
        return ans;
    }
};
```

#### 股票买卖含冷冻期

```
给定一个整数数组，其中第i个元素代表了第i天的股票价格

设计一个算法计算出最大利润。在满足以下约束条件下，你可以尽可能地完成更多的交易（多次买卖一支股票）:

你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。
卖出股票后，你无法在第二天买入股票 (即冷冻期为 1 天)。

输入: [1,2,3,0,2]
输出: 3 
解释: 对应的交易状态为: [买入, 卖出, 冷冻期, 买入, 卖出]
```

上述股票问题每个时刻可能有两个状态, 持有股票或者不持有股票。显然这次可以增加一个状态, 变成了某时刻末三个状态(或者说被决策所影响的三个状态), 0. 持有股票; 1. 未持有股票但上个时刻刚刚卖出股票(这个时刻在冷冻期); 2. 未持有股票且上个时刻也没有卖出股票(这个时刻不在冷冻期)。

找到状态的转移, 具体如下

```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        size_t n = prices.size();
        if (n <= 1)
            return 0;
        vector<vector<int>> f(n, vector<int>(3, 0));
        f[0][0] = -prices[0];
        f[0][1] = 0;
        f[0][2] = 0;

        for (int i = 1; i < n; i++) {
            f[i][0] = max(f[i-1][0], f[i-1][2]-prices[i]);  // 前一时刻持有股票f[i-1][0], 或前一时刻不持有股票当前时刻买了f[i-1][2]-prices[i](减了钱), 冷冻期d[i-1][i]不能买
            f[i][1] = f[i-1][0]+prices[i];  // 该时刻持有当前时刻卖了(加了钱), 进入冷冻期
            f[i][2] = max(f[i-1][1], f[i-1][2]);    // 非冷冻期非持有且不可能卖股票, 前一时刻卖了股票是冷冻期, 或者前一时刻不持有股票(当前时刻啥也没干, 和前时刻一样)
        }

        /// 两种不持有股票的最大值
        return max(f[n - 1][1], f[n - 1][2]);
    }
};
```

#### 最小堆
初始时堆为空。首先将最小的丑数 1 加入堆。

每次取出堆顶元素 x，则 x 是堆中最小的丑数，由于 2x, 3x, 5x 也是丑数，因此将 2x, 3x, 5x 加入堆。

上述做法会导致堆中出现重复元素的情况。为了避免重复元素，可以使用哈希集合去重，避免相同元素多次加入堆。

在排除重复元素的情况下，第 n 次从最小堆中取出的元素即为第 n 个丑数。

```cpp
class Solution {
public:
    int nthUglyNumber(int n) {
        priority_queue<long, vector<long>, greater<long>> prior_queue;
        prior_queue.push(1);
        set<long> numset;

        long result;
        for (int i = 0; i < n; i++) {
            result = prior_queue.top();
            prior_queue.pop();
            if (!numset.count(2*result)) {
                prior_queue.push(2*result);
                numset.insert(2*result);
            }

            if (!numset.count(3*result)) {
                numset.insert(3*result);
                prior_queue.push(3*result);
            }
                
            if (!numset.count(5*result)) {
                numset.insert(5*result);
                prior_queue.push(5*result);
            }
            
        }

        return result;
    }
};
```

虽然这个方法比较丑，但是暴力求解的绝佳。优先队列控制排序，set结构来防止重复, 二者结合就是排序的，没有重复的序列。

#### 基于动态规划的优化
定义数组dp[n], 表示前n个丑数
定义三个指针 p1, p2, p3, 分别dp数组的某个位置, 表示指向位置的丑数乘对应的质因数2,3,5为下次丑数的候选质因数。初始时，三个指针指向位置均为0.

下一个丑数可以认为是`min(dp[p1]*2, dp[p2]*3, dp[p3]*5)`, 但是要注意重复。也就是上一个丑数不等于`min(dp[p1]*2, dp[p2]*3, dp[p3]*5)`时说明下一个丑数有效, 同时将下一个丑数对应的指针p右移。

```cpp
class Solution {
public:
    int nthUglyNumber(int n) {

        vector<int> dp(n);
        dp[0] = 1;

        int p1 = 0, p2 = 0, p3 = 0; // 分别表示*2,*3,*5当前的因数对象
        int idx = 1;
        while(idx < n) {
            int temp = min(dp[p1]*2, min(dp[p2]*3, dp[p3]*5));
            if (temp != dp[idx-1]) {
                dp[idx]=temp;
                idx++;
            }
            if (dp[p1]*2 == temp) {
                p1++;
            }
            else if (dp[p2]*3 == temp)
                p2++;
            else if (dp[p3]*5 == temp)
                p3++;
        }

        return dp[n-1];
    }
};
```

类似的, leetcode313超级丑数
```
超级丑数 是一个正整数，并满足其所有质因数都出现在质数数组 primes 中。

给你一个整数 n 和一个整数数组 primes ，返回第 n 个 超级丑数 。

题目数据保证第 n 个 超级丑数 在 32-bit 带符号整数范围内。

```

对于素数列表primes size为N, 时间复杂度为O(n*N)
```cpp
int nthSuperUglyNumberDP(int n, vector<int>& primes) {

        vector<int> dp(n);
        dp[0] = 1;
        int numprimes = primes.size();
        vector<int> ptrs(numprimes); // 对应primes的指针
        int idx = 1;
        while(idx < n) { 
            int min_ptr = 0;
            long min_value = INT64_MAX;
            for (int i = 0; i < numprimes; i++) {
                long temp = dp[ptrs[i]]*primes[i];
                if (min_value > temp) {
                    min_value=temp;
                    min_ptr = i;
                }
            }
            if (min_value != dp[idx-1]) {
                dp[idx] = min_value;
                idx++;
            } // 有效
            ptrs[min_ptr]++;
        }

        return dp[n-1];
    }
```

以上做法, ptrs存储的是坐标, 每次求`dp[ptrs[i]]*primes[i]`的最小值, 这样有个问题, 大数乘法计算次数太多, 以至于整数乘法也很耗时, 事实上这个做法会超时。

解决办法是用一个数据记录下每个指针当前`dp[ptrs[i]]*primes[i]`的值, 使用时直接比较就可以了，这样可以减少大量重复的乘法计算。

```cpp
    int nthSuperUglyNumberV3(int n, vector<int>& primes) {
        vector<long> dp(n + 1);
        int numprimes = primes.size();
        vector<int> ptrs(numprimes, 0);
        vector<long> nums(numprimes, 1);    // 记录每个指针下当前乘积
        for (int i = 1; i <= n; i++) {
            long minNum = INT64_MAX;
            for (int j = 0; j < numprimes; j++) {
                minNum = min(minNum, nums[j]);
            }
            dp[i] = minNum;
            for (int j = 0; j < numprimes; j++) {
                if (nums[j] == minNum) {    // 更新所有==minNum的nums[j]
                    ptrs[j]++;
                    nums[j] = dp[ptrs[j]] * primes[j];  // dp素数位置*primes[j]
                }
            }
        }
        return dp[n];
    }
```

#### 环绕字符串中唯一的子字符串, 奇妙动态规划

```
把字符串 s 看作是“abcdefghijklmnopqrstuvwxyz”的无限环绕字符串，所以 s 看起来是这样的："...zabcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyzabcd....". 

现在我们有了另一个字符串 p 。你需要的是找出 s 中有多少个唯一的 p 的非空子串，尤其是当你的输入是字符串 p ，你需要输出字符串 s 中 p 的不同的非空子串的数目。 

输入: "zab"
输出: 6
解释: 在字符串 S 中有六个子串“z”、“a”、“b”、“za”、“ab”、“zab”。.
```

一种想法是计算字符串p以每个字符结尾的连续字符的长度, 例如`zab`, 以z, a, b结尾的连续字符长度分别为1,2,3, 所以子串长度为1+2+3=6。

原因在于，在不重复的情况下, 例如`za`连续子串数量为3, `zab`就考虑到b的子串, 即`b`, `ab`, `zab`。如果b不和前面的连续, 那dp[2]=1, 如果连续则dp[2]就是一直连续的字符串数量k。
```cpp
class Solution {
public:
    bool isContinous(char prev, char curr) {    // 判断是否连续
        if (prev == 'z') return curr == 'a';
        return prev + 1 == curr;
    }
    int findSubstringInWraproundString(string p) {
        vector<int> dp(26, 0);
        int N = p.size();
        int k = 0;
        for (int i = 0; i < N; ++i) {
            if (i > 0 && isContinous(p[i - 1], p[i])) {
                ++k;
            } else {
                k = 1;
            }
            dp[p[i] - 'a'] = max(dp[p[i] - 'a'], k);
        }
        return accumulate(dp.begin(), dp.end(), 0);
    }
};
```
