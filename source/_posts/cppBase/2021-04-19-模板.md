#### 函数模板
以下是一个函数模板，返回两个数之间的最大值。T是类型参数。
```cpp
template <typename T>
T max (T a, T b){
    return b < a ? a : b;
} 

int a = max(1, 2);  // 不必max<int>(1,1)
```
函数模板一般不用显示输入类型参数, 自动推导, 类型模板一般不可以

#### 两阶段编译检查
模板分两步检查，
1. 模板定义阶段，只进行类,函数是否定义的检查, 不包含模板类型和对象参数的检查。
2. 模板实例化阶段，模板会再次被检查，尤其是依赖于类型参数的形参。

```cpp
template<typename T>
void foo(T t){
    undeclare();    // 若函数未定义，第一阶段报错
    undeclare(t);   // 如果参数不对，第二阶段报错
}
```
以上可以总结为，和模板参数有关的，总是在第二阶段被检查。

#### 类型推断
1. 调用参数按引用传递，**任何类型转换都不允许**，传入的数组类型引用也不会退化, 通过模板类型T定义的两个参数，它们的实参必须完全一样。
2. 调用参数按值传递，引用和指针会退化，`const`和`volatile`限制符会被忽略。**引用转化成被引用的类型，raw array和函数被转换成相应的指针类型**。T定义的两个参数，实参退化后类型必须一样。

```cpp
template<typename T>
T max (T a, T b);

int& ir = i;
max(i, ir); // 被推导成int, 引用被decay
int arr[4];
foo(&i, arr);   // arr被推导成int*, 退化成指针
// 以下错误
max(4, 7.2);    // 错误，不确定被推导为int还是double
```

有两种办法解决推导歧义的错误
```cpp
1. 对参数做类型转换
max(static_cast<double>(4), 7.2);   
2. 显示指出参数T的类型，编译器不再进行类型推导
max<double> (4,7.2);
3. 设置不同模板参数
template<typename T， typename M>
T max (T a, M b);
max<int, double>(5, 7.2)
```
#### 默认调用参数
模板也可以有默认构造参数
```cpp
template<typename T>
void f(T = "");

f(1);   // ok,调用f<int> (1)
f();    // 错误，无法推断T的类型

// 应对这一情况，增加一个默认参数
template<typename T = std::string>
void f(T = "")
f();    // ok
```

#### 返回类型推断
C++11 中，尾值返回类型(trailing reture type) 用箭头表示声明返回类型。可以使用decltype推导表达式, decltype可能也是基于语法树推导返回类型, 推导表达式不要太复杂
```cpp
template<typename T1, typename T2>
auto max (T1 a, T2 b) -> decltype(b < a ? a : b)
{
    return b < a ? a : b;
}
```

需要注意，T1可能是引用因此会推导出返回值为引用，但我们想让它返回退化后的T
```cpp
#include <type_traits>
template<typename T1, typename T2>
auto max (T1 a, T2 b) -> typename std::decay<decltype(true ? a : b)> :: type
```

struct declay, Applies lvalue-to-rvalue, array-to-pointer, and function-to-pointer implicit conversions to the type T, removes cv-qualifiers, and defines the resulting type as the member typedef type, 数组会推导成指针。
```cpp
#include <type_traits>
 
template <typename T, typename U>
struct decay_equiv : 
    std::is_same<typename std::decay<T>::type, U>::type 
{};
template <typename T, typename U>
constexpr bool is_decay_equ = decay_equiv<T, U>::value;
 
int main()
{
    static_assert(
        is_decay_equ<int, int> &&
        ! is_decay_equ<int, float> &&
        is_decay_equ<int&, int> &&
        is_decay_equ<int&&, int> &&
        is_decay_equ<const int&, int> &&
        is_decay_equ<int[2], int*> &&
        ! is_decay_equ<int[4][2], int*> &&
        ! is_decay_equ<int[4][2], int**> &&
        is_decay_equ<int[4][2], int(*)[2]> &&
        is_decay_equ<int(int), int(*)(int)>
    );
}
```

#### 模板类的特例化

特化一个模板, 模板特化可以实现简单的模板函数接受不同的值, 而不用写大量类似的重载版本
```cpp
template<>
class Stack<std::string> {
    ...
};
```
部分特例化，例如可以特殊化一个Stack<>专门处理指针
```cpp
template<typename T>
class Stack<T*>{
};
```

#### 类型别名 Type Aliases
1. 使用关键字typedef
```cpp
typedef Stack<int> intStack;
void foo (intstack const& s);
```
2. 使用关键字using, using可以声明模板类型别名typedef 不可以
```cpp
using intStack = Stack<int>;
using add_const_t = typename add_const<T>::type;
```

#### 非模板类型参数

c++17开始，可用`auto`来设置任意有效的非类型模板参数
```cpp
#include <array>
#include <cassert>

template<typename T, auto Maxsize>
class Stack {
    public:
    using size_type = decltype<Maxsize>;
    private:
    std::array<T, Maxsize> elems;
    size_type numElems;
}
```

#### typename关键字用来声明类型
typename表示声明一个数据类型别名, 而不是定义一个对象
```cpp
typename T::subType* ptr

typename T::const_iterator pos;
```
以上`typename`用来澄清subType是定义在class T的一个类型，而不是数据成员。如果没有typename，subType会被假设成一个非类型成员。

#### 零初始化
对于基础类型，比如int,指针类型，由于它们没有默认构造函数，因此它们不会被初始化成一个有意义的值。
```cpp
int x;  // x has undefined value
int *ptr;   // ptr point to anywhere
```
对于内置类型，最好显示调用其默认构造函数初始化，比如
```cpp
template<typename T>
void foo()
{
    T x{};
}

template<typename T>
class MyClass {
private:
    T x;
public:
    MyClass() : x{}
    // ensures x is initizlized even for built-in tboidype
}
```

#### 继承
对于类模板，如果它的基类也依赖于模板参数，那么`this->x`和`x`不一定等效。
```cpp
template<typename T>
class Base {
public:
    void bar();
};
template<typename T>
class Derived : Base<T>{
public: 
    void foo() {
        bar();  // call external bar or error() 
    }
}
```
以上Derived中的bar永远不会被解析成Base中的bar,作为经验法则，建议用`this->`或`base<T>::`修饰之。


#### 值传递
当按值传递参数时，原则上所有的参数都会被拷贝，但注意被传递的参数是左值`lvalue`，拷贝构造函数会被调用。但是传递参数是右值，编译器会优化参数传递，使拷贝构造函数不会被调用。   

按值传递会导致类型退化decay，裸数组退化成指针，const和volatile等限制符被解除。

```cpp
template<typename T>
void printV (T arg){
    ...
}

std::string const c = "hi";
printV(c);  // c decays so that atg type is std::string
printV("hi"); // char const[3] decays to pointer, char const*
int arr[4];
printV(arr); // decays to pointer arg type int*
```

#### 引用传递
引用传递类型不会退化，也就是裸数组不会转换成指针，也不会移除`const`和`volatile`等限制符。
```cpp
template<typename T>
void printR(T const& arg){

}
std::string const c = "hi";
printR(c);  // 参数，std::string const&
int arr[4];
printR(arr);    // 参数，int const(&) [4]
```

#### 按转发引用传递参数
```cpp
template<typename T>
void passR (T && arg) {

}
std::string s = "hi";
passR(s);   // T std::string&, arg is also
passR(std::string("hi")); // T std::string, arg std::string&&

passR(std::move(s)); // T std::string arg is std::string&&
passR(arr); // T int(&)[4] also the type of arg
```
综上，**左值与通用引用放在一推导出来的T仍为左值，而右值与通用引用放在一起推导出来的T仍然为右值**

#### 动态多态
多态设计的艺术性主要体现在从一些相关的对象类型中提炼出一组统一的功能，然后将它们声明成一个基类的虚函数接口。而特定的功能则从它做了继承。
```cpp
#include "coord.hpp"

class GeoObj {
    public:
    virtual void draw() const = 0;
    virtual Coord center_of_gravity() const = 0;
    virtual ~GeoObj() = default;
};

class Circle : public GeoObj {
    public:
    virtual void draw() const override;
    virtual Coord center_of_gravity() const override;
};

class Line : public GeoObj {
    public:
    virtual void draw() const override;
    virtual Coord center_of_gravity() const override;
};

Coord distance (GeoObj const& x1, GeoObj const& x2)
// 传引用，可以保护x1,x2的动态类型
Line x1,x2;
distance(x1,x2); // distance(GeoObj&, GeoObj&)
```

#### 静态多态
不再使用虚函数层级，而是独立的几何类，如下
```cpp

class Circle{};
class Line{};

template <typename GeoObj>
void myDraw (GeoObj const& obj){
    obj.draw();
}

template<typename GeoObj1, typename GeoObj2>
Coord distance (GeoObj1 const& x1, GeoObj2 const& x2){}

Circle c1,c2;
distance(c1,c2); // distance<Circle, Circle>
Line l;
distance(l, c1); // distance<Line, Circle>(GeoObj1&, GeoObj2&)
```

通常静态多态比动态多态更加类型安全，因为其所有的绑定都在编译期进行了检查。

#### 模板编译单元的问题

模板跨编译单元链接可能出错, 因此必要时模板全部写在.hpp中, 不分.h和.cpp

如果你有三个文件, 分别是如下的组织形式
```cpp
// add.h
template <typename T>
T add(const T &a, const T &b);

// add.cc
#include "add.h"
template <typename T>
T add(const T &a, const T &b)
{
    return a + b;
}

// main.cc
#include "add.h"
int main()
{
    int i = add(1, 1);
    return 0;
}
```
然后你用`gcc -o main main.cc add.cc`编译链接, 发现出错
```
main.cc:(.text+0x34): undefined reference to `int add<int>(int const&, int const&)'
collect2: error: ld returned 1 exit status
```

相信能猜到这是链接的问题, 因为main.cc和add.cc是两个编译单元, 在链接时出现了不能链接`int add<int>(int const&, int const&)`, 因为模板的关系, 事实上并没有实例化`add<int>`

#### 虚函数不能设置成模板函数

同理，虚函数不能设置为模板函数(但可以使用类的模板)，因为对于虚函数来说, 它调用多少次需要运行时确定。因为`Base* base = new Drive;`条件下base会调用派生类的虚函数。**但对于模板来说, 它必须编译时确定函数被调用多少次, 并据此实例化几个不同类型的函数**。既然不能确定虚函数调用多少次，因此虚函数不能设置模板。

另外对于虚函数表来说，它的大小是固定的，换言之一个class对应一个虚函数表, 对于`Base* base = new Drive;`, base使用Drive class的虚函数表, 因此对于虚函数它会调用Derive的函数, 对非虚函数它还是使用Derive的表, 因为Derive对象的非虚函数继承自Base。但是如果虚函数是模板, 那岂不是模板类型为int实例化一个, float实例化一个, 虚函数表大小会大大膨胀, 这也不应该。

构造函数可以是模板参数, 析构函数没有参数, 因此也不存在模板情况。
```cpp
template <typename T>
class templateFunc
{
public:
	void fun() {
 
	}
	//ok:
	template<typename A>
	void testFunc(const A& a) {
		std::cout << "templateFunc::testFunc(const A& a) : " << a << std::endl;
	}
 
	//error: 编译器会直接报错
    /*
	template<typename A>
	virtual void testMe(const A& a) {
	}
    */

	virtual void testMe(const T& a) {   // 没问题
	}
};
```

#### 模板继承问题

这里涉及到模板的两阶段查找问题Two phase lookup, 准确的说
1. 在模板定义阶段，先对独立的(与模板参数无关的)成员进行检查，检查不通过则编译报错;
2. 在实例化模板时，对非独立的成员检查(检查是否在子类或基类中)

这样对于继承, 第一阶段查找会把父类和子类分别查找, 因此子类调用父类函数就可能编译出错(因为它不关心父类)。但如果第一阶段编译通过，第二阶段就把模板类作为非独立的检查一遍，这时候就会通过。

```cpp
template<typename T>
class A
{
    public:
        void add()
        {
            printf("A add\n");
        }
};

template<typename T>
class B:public A<T>
{
    public:
        void cal(){
            add(); // 报错, 这里第一阶段查找时，add()定性为独立，认为它是非成员变量，但又找不到
            this->add();// 此处正确，第一阶段认为它是非独立(因为与this有关，即与Ｔ有关), 会进行到第二阶段
			A<T>::add();	// 正确
		}
};
int main()
{
    B<float> b;
    b.cal();
}
```

似乎模板修饰class存在三个主要区别, 继承由于第一阶段检查可能编译不过, 虚函数不能设置为模板参数函数, 不同编译单元的模板可能延迟编译导致无法链接。其他无太大区别。
```cpp
template<typename T>
class innerBase
{
public:
    virtual void func() = 0;	// 纯虚函数

    virtual void func1() {	// 虚函数
         cout << "innerBase" <<"\n";
    };
protected:
    T a;
};

template <typename T>
class innerDrive : public innerBase<T>
{
public:
    virtual void func1() override{
         cout << "innerDrive" <<"\n";
    };
    virtual void func() override	// 重载
    {
        //此处必须使用innerBase::a（或者使用outer::innerBase::a）来访问a
        //否则无法找到a的定义
        cout << innerBase<T>::a << endl;
        //cout << a<<endl;    // 报错
        cout << this->a<<endl;
    }
};

int main() {
    innerDrive<int> d;
    d.func();

    innerBase<int> *ib = new innerDrive<int>;
    ib->func1();

    delete ib;

    return 0;
}

输出
0
0
innerDrive
```

类似的还有inline函数, 一般inline函数必须把定义放在.h文件中, 目的是所有用到inline函数的编译单元都有inline函数的定义, 而不是通过链接(inline函数必须编译在编译单元中因为它不基于函数调用)。链接意味着调用别的编译单元的函数必须要进行调用, 另外如果类成员函数在.h中定义, 那它也是inline的, 因此短的类成员函数也可以在.h中定义比较函数调用的损耗。


### 总结

函数模板自动推导参数, 类模板一般不行。

两阶段检查, 第一阶段不会检查参数, 继承依赖关系等。

类型推断, 模板参数值传递会数组退化, 去掉指针引用修饰符, const被忽略; 引用传递则保持原来的类型,数组也不会退化。

模板默认构造参数, 返回类型推断

模板类特例化, 灵活使用模板传参

typename声明类型, 不会产生对象创建歧义。

模板可以传入多种类型, 某种意义上可以是静态多态。静态多态只要通过编译运行时是安全的

模板多个编译单元链接时可能存在问题, 同样两个模板类有继承依赖关系时也可能存在问题
