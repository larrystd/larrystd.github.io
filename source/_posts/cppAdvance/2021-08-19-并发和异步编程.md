> 所有的并发都是被管理，有消息传递的局部并行。异步执行不意味着进程放任不管了, 至少需要进行在执行到某个地方返回有效的信息。

并发编程需要思考的是进程的协调，互相调用。

### 生产者消费者模型
这里使用`mutex`和`condition_variable`来协调线程。

主要记住,
* wait条件不满足之后会释放锁等待, 以后必须被其他线程唤醒才会向下执行, 而不是等待条件满足了。
* 使用条件变量, `wait`和`notify_all`要同时用, 且`notify_all`之前需要释放锁。这时候我们可以选择手动释放锁(`lock_guard`不能手动释放), 或用`{}`设置块作用域
* 为了能够缓慢复现, 一定给予sleep时间, 也就是`std::this_thread::sleep_for(dura)`

```cpp
#include <iostream>
#include <mutex>
#include <thread>
#include <condition_variable>
#include <queue>

using namespace std;

class ComsumerDemo {
public:
    void comsumer() {
        while (true) {
            //std::lock_guard<std::mutex> lcg(mtx);
            {
                std::unique_lock<std::mutex> lck(mtx);
                ctx.wait(lck, [this]() {    // 这里捕捉了this指针, 以便访问q.empty()
                    return !q.empty();  // 这里的谓词是wait until
                });
                //while (q.empty()) {
                //	ctx.wait(lck);	// wait unique_lock 而不是mutex
                //}
                int data = q.front();
                q.pop();
                printf("consume %d\n", data);
                cout << "consume q.size="<< q.size() <<endl;
            // 在这里unique_lock锁会自动释放
            }
            
            ctx.notify_all();
            /* 等待2S */
            // 一定留足时间
			std::chrono::milliseconds dura(2000);
			std::this_thread::sleep_for(dura);
        }      
    }
    void productor() {
        while (true) {

            std::unique_lock<std::mutex> lck(mtx);
            num = (num+1)%10;
            int n = q.size();
            ctx.wait(lck, [this, n]() {
                cout << "q.size="<< n <<endl;
                return n < 10;
            });

            //while (q.size() >= 10) {
            //    ctx.wait(lck);
            //}
            q.push(num);
            printf("produce %d\n", num);
            lck.unlock();   // 手动释放锁
            ctx.notify_all();

            /* 等待1S */
			std::chrono::milliseconds dura(1000);
			std::this_thread::sleep_for(dura);
        }
    }

    void start() {
        // 创建后自动执行函数
        std::thread t1(&ComsumerDemo::comsumer, this);
        std::thread t2(&ComsumerDemo::productor, this);
        //std::thread t3(&ComsumerDemo::comsumer, this);

        t1.join();
        t2.join();
        //t3.join();
    }
private:
    std::mutex mtx;
    std::condition_variable ctx;
    queue<int> q;
    int num;
};
```

### `future`, `promise`, `async`

`std::future`可以获取异步任务的结果，因此可以作为一种线程间同步的手段。`std::future`从结果出发, 一般会阻塞直到异步任务得到结果。

`std::future` 通常由某个 `Provider` 创建，Provider 可以被认为是异步任务的提供者。Provider 在某个线程中设置共享状态的值，与该共享状态相关联的 `std::future` 对象调用 get（通常在另外一个线程中） 获取该值，如果共享状态的标志不为 ready，则调用 `std::future::get` 会阻塞当前的调用者，直到 Provider 设置了共享状态的值（此时共享状态的标志变为 ready），`std::future::get` 返回异步任务的值或异常。这里需要注意`future`的作用是接收信息而不能写入信息, 传递信息是单向的。

std::future 对象通常由以下三种 Provider 创建，并和某个共享状态相关联。Provider 可以是函数或者类，分别是：

1. `std::async` 函数, 
2. `std::promise::get_future`，get_future 为 promise 类的成员函数，
3. `std::packaged_task::get_future`，此时 get_future为 packaged_task 的成员函数

#### async

std::async是异步操作做了一个很好的封装，使我们不用关注线程创建内部细节，就能方便的获取异步执行状态和结果，还可以指定线程创建策略。使用`std::async`如果nodejs的`async`, 创建一个线程异步执行, 主线程可以执行自己的任务, 以及调用get()等待结果的返回。async的构造方法
```cpp
template< class Function, class... Args>
std::future<std::result_of_t<std::decay_t<Function>(std::decay_t<Args>...)>>
    async( Function&& f, Args&&... args );

template< class Function, class... Args >
std::future<std::result_of_t<std::decay_t<Function>(std::decay_t<Args>...)>>
    async( std::launch policy, Function&& f, Args&&... args );  

// 第一个std::launch policy构造策略
```

第一个参数是线程的创建策略，有两种策略可供选择：

1. std::launch::async：在调用async就开始创建线程。
2. std::launch::deferred：延迟加载方式创建线程。调用async时不创建线程，直到调用了future的get或者wait时才创建线程。
默认策略是：`std::launch::async |std::launch::deferred`

然后对象`std::future`获取结果的方式有三种：
1. get：等待异步操作结束并返回结果
2. wait：等待异步操作结束，但没有返回值
3. waite_for：超时等待返回结果

```cpp
template <typename RandomIt>int parallel_sum(RandomIt beg, RandomIt end){
    auto len = end - beg;
    if (len < 1000)
        return std::accumulate(beg, end, 0);

    RandomIt mid = beg + len/2;
    auto handle = std::async(std::launch::async,
                             parallel_sum<RandomIt>, mid, end);
    int sum = parallel_sum(beg, mid);
    return sum + handle.get();  // 这里会等待async线程计算完
}

int main(){
    std::vector<int> v(10000, 1);
    std::cout << "The sum is " << parallel_sum(v.begin(), v.end()) << '\n';
}
```

#### promise

promise 对象作用是维护共享变量, 它可以保存某一类型 T 的值，该值可被 future 对象读取（可能在另外一个线程中），因此 promise 提供了一种线程同步的手段。相比`async`的主线程等待子线程的结果, promise的作用是把future给子线程从而主线程向子线程传递信息, 这和`async`的效果是相反的。可以认为`promise`和`future`是成对出现的, 对promise赋值, 可以通过future获取这个值

可以通过 get_future 来获取与该 promise 对象相关联的 future 对象，调用该函数之后，两个对象共享相同的共享状态(shared state)

promise 对象是异步 Provider，它可以在某一时刻设置共享状态的值。future 对象可以异步返回共享状态的值，或者阻塞调用者并等待共享状态标志变为 ready，然后才能获取共享状态的值。换言之, promise对象的set()必定`happen-before`promise对象的get()

```cpp
#include <iostream>       // std::cout
#include <functional>     // std::ref
#include <thread>         // std::thread
#include <future>         // std::promise, std::future

void print_int(std::future<int>& fut) {
    int x = fut.get(); // 获取共享状态的值.
    std::cout << "value: " << x << '\n'; // 打印 value: 10.
}

int main ()
{
    std::promise<int> prom; // 生成一个 std::promise<int> 对象.
    std::future<int> fut = prom.get_future(); // 获取一个future对象和 promise 关联.
    std::thread t(print_int, std::ref(fut)); // 将 future 交给另外一个线程t.
    prom.set_value(10); // prom设置共享状态的值, 此处和线程t保持同步, 线程t通过future获得
    t.join();
    return 0;
}
```

### 异步等待模型

使用`future`可以轻易实现异步等待。即线程2等待线程1执行后立即执行，依次类推。

原因在于`std::future<T>::wait` 和 `std::future<T>::get` 都会阻塞地等待拥有它的 `promise` 对象返回其所存储的值，后者还会获取 T 类型的对象;

```cpp
class Foo {
    promise<void> pro1, pro2;

public:
    void first(function<void()> printFirst) {
        printFirst();
        pro1.set_value();
    }

    void second(function<void()> printSecond) {
        /// 下面这句话相当于等待pro1执行完set_value()再运行
        pro1.get_future().wait();
        printSecond();
        pro2.set_value();
    }

    void third(function<void()> printThird) {
        pro2.get_future().wait();
        printThird();
    }

    void start() {
        std::thread t1 (&Foo2::first, this,[]() {
            cout << "first" << endl;
        });

        std::thread t2 (&Foo2::second, this,[]() {
            cout << "second" << endl;
        });

        std::thread t3 (&Foo2::third, this,[]() {
            cout << "third" << endl;
        });

        t1.join();
        t2.join();
        t3.join();
    }
};
```

### 原子模型

将共享标志变量设置为原子变量，从而在一个线程执行完修改标志位，使另一个线程执行。原子变量也有happen-before语义, 即`store`happen-before`load`

```cpp
class Foo {
    std::atomic<bool> a{ false };
    std::atomic<bool> b{ false };
public:
    void first(function<void()> printFirst) {
        printFirst();
        /// 修改标识变量
        //a = true;
        a.store(true);
    }

    void second(function<void()> printSecond) {
        /// 可以直接循环等待标识变量的修改
        while (!a)
            this_thread::sleep_for(chrono::milliseconds(1));
        printSecond();
        //b = true;
        b.store(true);
    }

    void third(function<void()> printThird) {
        while (!b.load())
            this_thread::sleep_for(chrono::milliseconds(1));
        printThird();
    }
};
```
