### 图的存储

#### 单链表

* 存储每个点的值`e[i]`
* 存储每个结点的下一个结点`ne[i]`
* 存储头结点`head`

插入节点

```cpp
void add_to_head(int x) // 头插法节点, 降序排列
{
    // idx这里是新id, 表示节点的id
	e[idx] = x; // 1. 存储节点的值
	ne[idx] = head; // 2. 新结点ne指向当前头结点, 头插法
	head = idx; // 3. head指向新结点
	idx ++ ; // 4. 节点个数+1
	
    /* idx实际上记录了每个点是第几个输入的点 */
}
```

### 拓扑排序 

拓扑排序可以判断图中是否存在环

#### Leetcode 207 课程表

```
你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1 。

在选修某些课程之前需要一些先修课程。 先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则 必须 先学习课程  bi 。
 
    例如，先修课程对 [0, 1] 表示：想要学习课程 0 ，你需要先完成课程 1 。

请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。
```

要求我们判定图中是否存在环，或者拓扑排序原理，可以使用bfs或者dfs进行。

一些小trick,
1. 基于vector来描述图的邻接表，
```cpp
vector<int> visited(numCourses);    // 长为numCourses的vector，初始化为0
vector<vector<int>> adj(numCourses);    // 形成长为numCourses的vector，不会初始化
for (const auto& info: prerequisites) {
    adj[info[1]].push_back(info[0]);    // 可以看成邻接表
}
```

2. 递归时设置合理的全局变量，可以提高清晰性。
3. 分析递归代码，看递归了几层，每层卡在哪个地方，当前主要变量值是什么。

* dfs求解

主要思路
1. 合理生成邻接表
2. 从未访问开始进行dfs。如果存在环，则从环上每个节点遍历都能判定环。例如有a,b,c点，**从a点dfs访问到b，未发现环，则之后也没必要再以b为起点进行访问了。但注意以c为起点进行dfs时是可以访问b的**。
3. visited需要有三个标注，visited=0表示没有访问过，visited=1表示当前dfs访问到的，一旦dfs到visited=1说明有环，visited=2表示已经访问完的。visited=1和visited2的区别就是dfs访问到visited=1的节点说明有环。

```cpp
class Solution {
public:
    bool valid = true;

    void dfs(int start, vector<vector<int>>& adj, vector<int>& visited){

        visited[start] = 1;

        for (int e : adj[start]){
            if (visited[e] == 0){   // =0说明可以访问
                dfs(e,adj,visited);
                if (!valid)
                    return;
            }else if (visited[e] == 1){ // 访问到=1的节点说明有环，也就是访问到在栈里面的元素
                valid = false;
                return;
            }
        }
        visited[start] = 2; // =2与1区别，已经访问完递归回来的，或已经出栈的    
    }


    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        vector<int> visited;

        vector<vector<int>> adj;

        visited.resize(numCourses);
        adj.resize(numCourses);
        
        for (const auto& info: prerequisites) {
            adj[info[1]].push_back(info[0]);
        }
        
        for (int i = 0; i < numCourses && valid; ++i) {
            if (!visited[i]) {
                dfs(i,adj,visited);
            }
        }
        return valid;
    }
};
```

* bfs求解

bfs求解拓扑排序是一般的解法，基本步骤如下
1. 选取入度为0的顶点，全部入队
2. 队首元素出队，并删除队首元素连接的边，也就是队首连接的元素入度减1
3. 对队首元素，使入度为0的节点入队

经过以上步骤，**倘若存在环，环中的元素不可能入度为零节点**。因此只需要判定入过队的元素数量是否为全部节点，即可判定是否存在环。

```cpp
class Solution {
public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        vector<vector<int>> adj(numCourses);//邻接表
        vector<int> inDegree(numCourses);    //入度

        for (vector<int> v : prerequisites){
            adj[v[1]].push_back(v[0]);    // x[1]->x[0]
            inDegree[v[0]]++;
        }
        queue<int> q;
        // 找一个入度为0的点
        for (int i = 0; i < numCourses; i++){
            if (inDegree[i] == 0){
                q.push(i);
            }   
        }
        int count = 0;
        while (!q.empty()){
            int f = q.front();
            q.pop();
            count++;
            for (int k : adj[f]){
                inDegree[k]--;
                if (!inDegree[k])
                    q.push(k);
            }         
        }
        return count == numCourses;
    }
};
```

1. 使用vector模拟的邻接表处理图更方便
2. 分析dfs时注意递归层数，每层卡在哪里，主要变量值是什么
3. bfs处理搜索问题有时效果很好，而且清晰易懂，编程不易出错。

### 两点之间的最短路

#### 所有边权相同的情况

当边没有权值或权值相同时，最短路问题可以简单通过BFS解决, BFS一层迭代, 代表路径+1。

```cpp
int bfs()
{
    queue<pair<int,int>> que; //pair记录城市编号和dis，也可以用结构体
    que.push({1,0}); //把起始点加入队列
    book[1] = 1; //标记为已在路径中
    while(!que.empty()) 
    {
        int cur = que.front();
        que.pop();
        for(int i = 1; i <= n; i++)
        {
            if(e[cur][i] != MAX && book[i] == 0) //若从cur到i可达且i不在队列中，i入队
            {
                que.push({i, cur.second+1});    // second+1表示路径+1
                book[i] = 1;    // 标记
                if(i == n) 
                    return cur.second; //如果已扩展出目标结点了，返回中转城市数答案
            }
        }
    }
}
```

#### 权值不同的情况
可以用DFS回溯的办法

```cpp
void dfs(int cur, int dis) //cur-当前所在城市编号，dis-当前已走过的路径
{
    if(dis > min) return; //若当前路径已比之前找到的最短路大，没必要继续尝试
    if(cur == n) //当前已到达目的城市，更新min
    {
        if(dis < min) min = dis;
        return;
    }
    
    for(int i = 1; i <= n; i++) //对1~n号城市依次尝试
    {
        if(e[cur][i] != INF && book[i] == 0) //若cur与i可达，且i没有在已走过的路径中
        {
            book[i] = 1; //标记i为已在路径中
            dfs(i, dis+e[cur][i]); //继续搜索
            book[i] = 0; //对从i出发的路径探索完毕，取消标记
        }
    }
}
```
### 任意两点的最短路

#### 单源最短路 ,即指定源点，求它到其余各个结点的最短路。

Dijkstra算法用来解决权值为正的不同权值边之间的最短路，Dijkstra 算法每决定移动一个格子都会计算它离起点所需要的代价`g(n)`, 同时选择选择最小代价的节点。整个算法的基本步骤是：

1. 所有结点分为两部分：已确定最短路的结点集合P、未知最短路的结点集合Q。最开始，P中只有源点这一个结点。（可用一个book数组来维护是否在P中）
2. 循环n-1次, 每次都在**Q中选取一个离源点最近的结点u**（`dis[u]`最小）加入集合P。然后**考察u的所有出边，做松弛操作**。注意每次找到Q设置Q为已知,也就是移出集合Q

```cpp
// 初始化dis
for(int i = 1; i <= n; i++) 
    dis[i] = e[1][i]; //初始化dis为源点到各点的距离

// book 记录已经访问的节点
for(int i = 1; i <= n; i++) 
    book[i] = 0; 
book[1] = 1; //初始时P集合中只有源点


/// 遍历n-1次,每次可以得到一个最短距离的点，再根据这个点进行松弛
for(int i = 1; i <= n-1; i++)
{
    int min = INF;
    int u = ;

    for(int j = 1; j <= n; j++) //找出未访问的, 寻找Q中最近的结点
    {
        if(book[j] == 0 && dis[j] < min)
        {
            min = dis[j];
            u = j;  // u是要选择的点
        }
    }
    book[u] = 1; //加入到P集合
    /// 基于u对n个点松弛操作
    for(int v = 1; v <= n; v++) //对u的所有出边进行松弛
    {
        if(e[u][v] < INF) 
        {
            if(dis[v] > dis[u] + e[u][v]) 
                dis[v] = dis[u] + e[u][v];
        }
    }
}
```

leetcode 743 网络延迟时间
```
有 n 个网络节点，标记为1到 n。

给你一个列表times，表示信号经过 有向 边的传递时间。times[i] = (ui, vi, wi)，其中 ui 是源节点，vi 是目标节点， wi是一个信号从源节点传递到目标节点的时间。

现在，从某个节点 K 发出一个信号。需要多久才能使所有节点都收到信号？如果不能使所有节点收到信号，返回 -1 。
```

* 即求最短路的最大值, 即所有节点都收到信号的最短时间

```cpp
    const int INF = INT32_MAX/2;
    int networkDelayTime_dij(vector<vector<int>>& times, int n, int k) {
        vector<bool> book(n+1, false);
        vector<int> dis(n+1, INF);

        /// 邻接矩阵建立
        vector<vector<int>> graph(n+1,vector<int>(n+1,INF));
        for(int i=1;i<=n;i++)    graph[i][i]=0;
        for(auto e:times)    graph[e[0]][e[1]]=e[2];

        for (int i = 1; i <= n; i++) {
            if (graph[k][i] != INF) {
                dis[i] = graph[k][i];
            }
        }
        book[k] = true;


        for(int i=1;i<n;i++){   // 遍历n-1个节点
            int min_id=0,min_dis=INF;
            for(int j=1;j<=n;j++){
                if(book[j]==false && dis[j]<min_dis){
                    min_dis=dis[j];
                    min_id=j;
                }
            }
            
            book[min_id]=true;
            // 松弛
            for(int j=1;j<=n;j++){
                dis[j] = min(dis[j], dis[min_id]+graph[min_id][j]);
            }
        }

        int ans = 0;
        for (int i = 1; i <= n; i++) {
            if (dis[i] == INF)
                return -1;
            ans = max(dis[i], ans);
        }
        return ans;
    }
```

#### 可以处理负权边的`Bellman-Ford`算法

* 可以用用三个变量存储边, 分别表示起始点,终止点,权重;例如第i条边表示`v[i][0]`->`v[i][1]`，权值为`v[i][2]`

* dis数组表示到每个节点的距离。外层循环执行n-1轮(因为最慢从源点慢慢延申也需要n-1次才能延申到所有顶点), 内层循环枚举每一条边, 根据边的两个顶点, 尝试更新`dis[edge[0]] = min(dis[edge[0]], dis[edge[1]] + edge[2])`。

* 和Dijkstra算法不同，前者基于贪心, 遍历n-1个点看能否松弛距离，后者是遍历每次可能的中间点。

```cpp
for(int i = 1; i <= n; i++) dis[i] = INF;
dis[1] = 0; //初始化dis数组，只有1号的距离为0

for(int k = 1; k <= n-1; k++) //进行n-1轮松弛
// 每一轮表示可以经过k次中间点
    for (auto&& flight: flights) {
        // 枚举每一条边
        int j = flight[0], i = flight[1], cost = flight[2];
        dis[i] = min(dis[i], dis[j] + cost);
    }
```

leetcode 787 K 站中转内最便宜的航班

```
有 n 个城市通过一些航班连接。给你一个数组 flights ，其中 flights[i] = [fromi, toi, pricei] ，表示该航班都从城市 fromi 开始，以价格 pricei 抵达 toi。

现在给定所有的城市和航班，以及出发城市 src 和目的地 dst，你的任务是找到出一条最多经过 k 站中转的路线，使得从 src 到 dst 的 价格最便宜 ，并返回该价格。 如果不存在这样的路线，则输出 -1。
```

* K站也就是中间点的个数，这正好是Bellman-Ford算法的外层循环。

```cpp
class Solution {
private:
    const int INF = 10000 * 101 + 1;

public:
    int findCheapestPrice(int n, vector<vector<int>>& flights, int src, int dst, int k) {
        vector<vector<int>> f(k + 2, vector<int>(n, INF));
        // 最多k次中转, 则意味着k+1次航班
        /// f[i][j] 表示i 次航班, 到src的最小花费
        /// 无中转i=1, k次中转i=k+1
        f[0][src] = 0;
        for (int t = 1; t <= k + 1; ++t) {
            for (auto&& flight: flights) {
                // 枚举每一条边
                int j = flight[0], i = flight[1], cost = flight[2];
                f[t][i] = min(f[t][i], f[t - 1][j] + cost);
            }
        }
        int ans = INF;
        for (int t = 1; t <= k + 1; ++t) {
            ans = min(ans, f[t][dst]);
        }
        return (ans == INF ? -1 : ans);
    }
};
```

### A*算法

* 令`g(n)` 代表移动到第 n 个方格时距离起点的代价。`g(n)` 的大小跟之前的 n-1 个格子的选择有关的，每次选择都贪心的选择代价最小的格子
* `h(n)` 代表移动到第 n 个方格时距离终点的估计代价(不知道确切代价), 每次选择`g(n)`也是贪心的选择代价最小的格子
* `f(n)` 是整体的估价函数，一般 `f(n)=g(n)+h(n)`。

一般会用两个表来记录整个找路径的过程，一个称之为 open 表（用于存储可能的选择），另外一个称之为 closed 表（用于存储已经做的选择）：

* 如果 open 表为空，表示上一步没有留下可以被选择的节点，则问题无解，退出搜索；
* 把 open 表的第一个节点取出来放入 closed 表，并记该节点为 n, 判断节点是否为所求是则退出; 否则扩展节点 n(即查看这个格子周围的格子), 加入到open表中，对open表中节点排序。之后再把第一个节点取出放入closed表,循环之。

这是一种启发式搜索(Heuristically Search), 或者说信息搜索(Informed Search)，该算法利用问题的启发信息(所求解问题相关的辅助信息)引导搜索过程，来减少搜索范围，降低问题复杂度。
