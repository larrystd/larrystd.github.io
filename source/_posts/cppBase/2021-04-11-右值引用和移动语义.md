### 右值引用
右值引用是C++11增加的主要新特性之一了，比较绕不容易理解，这里整理一下。   

#### 左值和右值
维基百科的定义为，C++表达式的"值分类"(value categories)属性为左值或右值。左值是可寻址的变量, 有持久性; 右值一般是不可寻址的常量(例如宏定义), 或在表达式求值过程中创建的无名临时对象, 短暂性的。**右值一般是字面量，临时对象，表达式等**, 字面量一般就是整型, 浮点型, 字符型, 字符串字面量。

能否被赋值不是区分C++左值与右值的依据，C++的const左值是不可赋值的；而作为临时对象的右值可能允许被赋值。**左值与右值的根本区别是能否通过&运算符获得对应的内存地址**，左值是实际存在内存中的对象(例如a=4;b=4;内存中两个地址存的值都是4), 右值字面量在编译后以左值对象的形式存在内存中或者只存在编译的替换中(没有对象引用这个字面量), 右值临时对象会在内存中创建, 但生存期很短且不能执行运算。

在C++03中, 右值临时对象是设置不可修改的, 可以用const T&接收; C++11 使用了右值引用T&&支持了右值的修改。右值引用一个优势是传值时处理右值临时对象的优势(避免拷贝)。

右值引用的第二个主要优势是构造对象传右值引用可以调用移动构造函数，支持内部指针对指向对象的所有权转移来加快构造的效率(即把当前指针指向对象给别人，当前指针设为nullptr, 类似auto_ptr), 基于左值的构造默认采用深拷贝。(如果你想深拷贝就拷贝构造, 如果要浅拷贝或所有权转移拷贝就用移动构造)

C++的表达式不是左值(lvalue)，就是右值(rvalue)。注意到C++中字面值和变量是最简单的表达式，其结果就是字面值和变量的值。

```cpp
int i = 0, j = 0, k = 0;	//初始化而非赋值
const int ci = i;	//初始化而非赋值

1024 = k;	//错误：字面值是右值
i + j = k;	//错误：算术表达式是右值
ci = k;		//错误：ci是常量左值，不可修改
```

**一般而言，左值表达式表示一个对象的身份，右值表达式表示的是对象的值**，不同的运算符对运算对象的要求各不相同，有的需要左值运算对象，有的需要右值运算对象;返回值也有差异，有的得到左值结果，有的得到右值结果。 **左值不能绑定到要求转换的表达式、字面常量或者返回右值的表达式**，但可以将右值引用绑定到这些表达式上。注意到右值虽然无法获取地址，但是右值引用是可以获取地址的，该地址表示临时对象的存储位置。

```cpp
int i = 42; // 初始化，声明+定义+赋值
int &r = i; // 引用，左值
int &&rr = i;   // 错误，不能将右值引用绑定到左值上
int &r2 = i * 42;   // 错误，i*42是一个右值表达式,左值引用不能绑定右值
const int &r3 = i * 42; // 正确，可以将一个const左值引用绑定到一个右值上
int &&rr2 = i * 42; // 正确，绑定到右值表达式上
```

**右值引用本身是一个左值**, 可以获取地址，**左值引用虽然不能绑定右值，但能绑定右值引用**
```cpp
int &&r = 10;  
int &l = r;
l = 11;
cout << r << endl;  // 11
```
另一种理解左值右值的方法是通过寄存器和内存，比如**常数5，在使用时不会在内存分配空间，而是直接放到寄存器中**，所以它在C++中就是一个右值。而定义一个变量 a，它在内存中会分配空间，在C++中就是左值。至于a+5，因为a+5的结果存放在寄存器中，它并没有在内存中分配新空间，所以它是右值。

```cpp
int &&rr3 = std::move(rr1);
```
move调用告诉编译器，我们有一个左值，但希望像右值一样使用它。这意味着承诺除了对rr1赋值或销毁之外，不再使用它。

#### 移动构造函数
移动构造函数和拷贝构造函数一样, 是构造函数的方法。只是传入左值或左值引用调用拷贝构造函数, 传入右值引用或右值调用移动构造函数。

拷贝构造函数默认遵循的实现规则是深拷贝, 移动构造函数则基于浅拷贝或所有权转移。因此移动构造函数构造对象往往更高。
```cpp
class A {
public:
    A(int ){
        cout << "constructor" << "\n";
    }
    A(A &&o) {
        cout << "moving constructor" << endl;
    }
    A(const A& o) {
        cout << "copy constructor" << endl;
    }
};

cout << "construct\n";
A a(10);
A& aref = a;
cout << "copy construct passing lvalue\n";
A a1(a);
cout << "copy construct passing lvalue reference\n";
A a2(aref);

// 输出，传左值或左值引用全部调用拷贝构造函数
construct
constructor
copy construct passing lvalue
copy constructor
copy construct passing lvalue reference
copy constructor
```

当然传右值或者右值引用会调用移动构造函数, 且移动构造函数效率更高。但可惜的是右值临时变量我们无法自己定义, 右值字面量只有整型, 字符串等少量类型。对上面的A对象，我们似乎只能手动定义左值或左值引用，那我们怎样应用效率更高的移动构造呢?

答案很简单, 让编译器把我们的左值或左值引用转为右值, std::move随之而来。
```cpp
cout << "move construct passing std::move of left value\n";
A a3(std::move(a));
cout << "move construct passing std::move of left reference value\n";
A a4(std::move(aref));

move construct passing std::move of left value
moving constructor
move construct passing std::move of left reference value
moving constructor
```

所有的这一切都是为了应用效率更高的移动构造函数来构造对象。(如果不提供移动构造函数, 就会自动调用拷贝构造)

注意我们是用左值或左值引用构造右值, 然后调用移动构造函数构造新对象。**移动构造之后我们原来的左值就不能用了!!!**, 这是运用移动构造必须考虑的，必须要小心。

移动构造需要设置不会抛出异常(std::swap), 这意味着移动构造应该要么不做要么成功。事实上移动构造不需要额外分配内存, 相比于拷贝构造不容易发生异常。
```cpp
StrVec::StrVec(StrVec &&s) noexcept
    : elements(s.elements), first_free(s.first_free)
{
    s.elements = s.first_free = nullptr;
}
```

#### std::move 实现
以上说明std::move就是一个类型转换器，将左值或左值引用转换成右值(引用), 因为对一个对象我们往往难以直接生成右值。
```cpp
template <typename T>
typename remove_reference<T>::type&& move(T&& t)
{
    return static_case<typename remove_reference<T>::type&&>(t);
}
```
move的输入参数类型称为通用引用类型。所谓通用引用，就是auto或者模板修饰的引用，可以接受左值引用和右值引用, 分别推导成对应的引用。实际上auto就是模板中的T，它们是等价的。之所以能接受左值引用和右值引用，是通过模板推导实现的。

模板推导时数组是不会退化成指针的, 所谓退化成指针可以理解为不能直接获得数组的长度了。这可以从C++具有静态类型反射的能力理解。
```cpp
template<typename T>
void f(T&& param){
    std::cout << "the value is "<< param << std::endl;
}

int main(int argc, char *argv[]){

    int a = 123;
    auto && b = 5;   // 通用引用，可以接收右值
    auto && d = a;   // 通用引用，可以接收左值
    int && c = a;    // 错误，右值引用，不能接收左值
    const auto && e = a;  // 错误，加了const另当别论
    // const&可以接收右值, 一般不使用const&&(因为没啥意义)
    func(a);         // 通用引用，可以接收左值
    func(10);        // 通用引用，可以接收右值
}
```

#### 参数模板推导
参数模板类型推导分成两大类：非引用/指针类型模板为一类; 引用和指针模板为另一类。

对于第一类其推导时根据的原则是，函数参数传值不影响原值，所以无论你实际传入的参数是普通变量、常量还是引用，它最终都退化为不带任何修修饰的原始类型。例如，`const int &`类型传如一个`T`的参数，退化为int型了。因此参数类型是拷贝构造, 不影响const, &等修饰的变量。

第二类为模板类型为引用(包括左值引用和右值引用)或指针模板。这一类在类型推导时根据的原则是去除对等数量的引用符号, 结构就是保持传入的语义。例如，`func(T& x)`中当x的类型为 `int&`，它与`T&`放在一起可以知道T为`int`, 但参数类型是int&。另一个例子function(T&& x)，其中x为`int&`它与`T&&` 放在一起可知T为`int&`, 当x为int&&则T推导成int, 参数类型变为int&&

这里使用了引用折叠的技术, 引用折叠是编译器行为, 例如`int&& &`折叠成int&
```
`int& &` 折叠为 `int&`
`int& &&` 折叠为 `int&`
`int&& &` 折叠为 `int&`
`int&& &&` 折叠为 `int &&`
```

#### 通用引用
通用引用是一种特别的引用, 体现在它可以接收左右值和左右值引用, 将它们转为合适的引用类型。自动进行了重载, 避免写代码时写多个参数重载的函数。
```cpp
template<typename T>
void f(T&& param){
    std::cout << "the value is "<< param << std::endl;
}
```
传入类型T&&，又因是模板可以进行类型推导，所以是通用引用，因此给它传左值和右值它都能正确接收。**左值与通用引用放在一推导出来的T仍为左值，而右值与通用引用放在一起推导出来的T仍然为右值**。

在C++11中，增加了一种成员称为**类型成员**。类型成员与静态成员一样，它们都属于类而不属于对象，访问它时也与访问静态成员一样用::访问。例如：
```cpp
typename remove_reference<T>::type&&
typedef std::string::size_type type_s;

template <typename T>
typename T::value_type top(const T &c){}    // 返回一个T::value_type类型
```
`remove_reference`类的实现, 通过模板特例化remove_reference可以接收T, T&, T&&, 内部的type成员都会获得T类型, 这一切都是编译器行为。
```cpp
typename remove_reference<T>::type&&

template <typename T>
struct remove_reference{
    typedef T type;  //定义T的类型别名为type
};

template <typename T>
struct remove_reference<T&> //左值引用
{
    typedef T type;
}

template <typename T>
struct remove_reference<T&&> //右值引用
{
   typedef T type;
}
```

#### 完美转发 perfectly forward arguments

通过`move`函数，我们可以将左值/左值引用转化成右值引用, 从而方便调用移动构造函数提高效率。(当然你的移动构造函数要基于移动写, 不是复制一遍拷贝构造函数)

forward需要传入模板参数T
```cpp
  template<typename _Tp>
    constexpr _Tp&&
    forward(typename std::remove_reference<_Tp>::type& __t) noexcept
    { return static_cast<_Tp&&>(__t); }
```

某些函数需要将其实参连同类型不变的转发给其他函数，而不再是无条件转化成右值，这时候就用到完美转发。完美转发似乎像一个设计模式, 往往存在通用引用模板函数中。总所周知通用引用将传入的类型转为适当的引用, 如果在通用引用函数内部我们想用到传入本身的类型就会用到完美转发。这时候通用引用的目的是少写几个重载函数
```cpp
#include <iostream>
using namespace std;
//重载被调用函数，查看完美转发的效果
void otherdef(int& t) {
    cout << "lvalue\n";
}
void otherdef(int&& t) {
    cout << "rrvalue\n";
}
void otherdef(const int & t) {
    cout << "rvalue\n";
}
//实现完美转发的函数模板
template <typename T>
void function(T&& t) {
    otherdef(t);
}
template <typename T>
void functionMove(T&& t) {
    otherdef(std::move(t));
}
template <typename T>
void functionForward(T&& t) {
    otherdef(std::forward<T>(t));  // t为右值时转发到int&&, 而不是const int&, 前者优先级更高
}
int main()
{
    function(5);
    functionMove(5);
    functionForward(5);
    int  x = 1;
    function(x);
    functionMove(x);
    functionForward(x);
    return 0;
}

// 输出
lvalue
rrvalue
rrvalue
lvalue
rrvalue
lvalue
```

std::forward属于较深入的模板编程了, 上升到设计模式的层次了, 因为一般不会使用

#### emplace_back和push_back
vector emplace_back部分使用了移动语义原理, 一个参数类型是`const value_type&`, 一个是`_Args&&... _args)`

```cpp
      void
      push_back(const value_type& __x)
      {
	if (this->_M_impl._M_finish != this->_M_impl._M_end_of_storage)
	  {
	    _Alloc_traits::construct(this->_M_impl, this->_M_impl._M_finish,
				     __x);
	    ++this->_M_impl._M_finish;
	  }
	else
	  _M_realloc_insert(end(), __x);
      }

#if __cplusplus >= 201103L
      void
      push_back(value_type&& __x)
      { emplace_back(std::move(__x)); }

#if __cplusplus >= 201103L
  template<typename _Tp, typename _Alloc>
    template<typename... _Args>
      void
      vector<_Tp, _Alloc>::
      emplace_back(_Args&&... __args)
      {
	if (this->_M_impl._M_finish != this->_M_impl._M_end_of_storage)
	  {
	    _Alloc_traits::construct(this->_M_impl, this->_M_impl._M_finish,
				     std::forward<_Args>(__args)...);
	    ++this->_M_impl._M_finish;
	  }
	else
	  _M_emplace_back_aux(std::forward<_Args>(__args)...);
      }
#endif
```

测试
```cpp

    cout << "******push next\n";
    vec.push_back(std::move(a));

    vec.push_back(a);
    cout << "******next\n";
    vec.push_back(10);
    return 0;
}

******emplace next
moving constructor
******next
copy constructor
copy constructor
******next
constructor
copy constructor
copy constructor
******push next
moving constructor
******next
copy constructor
copy constructor
copy constructor
copy constructor
copy constructor
******next

moving constructor
```
总结, 在C++11以上emplace_back和push_back处理左值时效率差距最大, 这体现在emplace_back原地构造的优越性上

传右值时push_back多执行一次构造函数, 多出来的是原地构造

传入std::move右值引用时两者都只执行一次移动构造, 因为push_back内部调用`emplace_back(std::move(__x));`

综上, 尽量用emplace_back和emplace, 当然emplace之后的对象就只在容器中使用吧

### 总结
左值和右值, 区别在于左值是已经定义的变量，持久性的存在内存中。右值字面量不直接存在内存中，右值临时对象只是暂时存在内存中。

右值引用的效率优势主要体现在右值临时对象上, 其一是传值时避免拷贝, 其二是构造时自动调用移动构造而非拷贝构造可以利用移动构造所有权转让的优势。移动构造默认采用所有权机制构造对象, 较基于深拷贝的拷贝构造效率更高

只有传入右值或右值引用才能利用移动构造优势, 但一般对象难以直接构造右值, 因而先构造左值再使用move将左值/左值引用转为右值引用从而可以利用移动构造。

由于移动构造基于所有权转移, 因而被move的左值对象不能再使用了, 析构函数也要注意(析构前判断指针是否为空)

例如，vector的emplace_back除移动语义外可以原地构造对象(不涉及右值引用原理); 这令emplace_back传入左值时较push_back有巨大优势(少几层拷贝)
