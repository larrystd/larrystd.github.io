> C++一项重要的设计规则是封装不降低运行效率, 模板元编程旨在编译期进行类型，变量的检查, 更充分的编译优化, 从而降低运行代码的复杂性, 提高运行效率。基于模板元将复杂的抽象代码编译成如C语言编译单元般简洁的运行代码, 又不失抽象, 提供强大的编译期检查保证, 代价是编译过程复杂且耗时。

### 模板元

* 模板编程可以同时操作类型和变量, 这很强大。注意`A<int>`, `A<0>`, `A<1>`是三种不同的类型, `<>`内容也是类型的一部分。
* 尽管可以有类模板和函数模板, 但模板元编程实现的单位往往是类型。因为函数模板支持参数类型自动推导, 往往只管函数传参就行了不需要设置`<>`(比如用的`sort(), lower_bower()`等函数, 其实也是模板函数但不需要设置类型); 但类模板要明确给出参数类型才可以

使用枚举类的目的是实现`value=A+B`的计算, enum内的变量没有内部的域, 换言之仍是`tyPlus::value`可以访问的。注意`enum class`有内部域, 访问通过`tyPlus::t::value`。

注意cout有坑, 类必须重载了`operator<<`才能用cout, 但在模板编程中类型比较多且复杂有可能可以打印但没有重载`operat<<`造成编译错误, 这个需要注意。

#### 元编程

模板编程, 目的是实现模板匹配(匹配所有`<>`的内容)和实例化(语义合理)。

#### 函数模板的元编程
decltype(A+B)可以推导类型
```cpp
template <int A, int B>	// 能匹配<int A, int B>即完成版本匹配, 匹配后再实例化且语义正确。
auto tyPlus2() -> decltype(A+B) {
    return A + B;
};

int main() {
    printf("%d\n",  tyPlus2<2,3>());
}

5
```

#### 模板特化的元编程

模板匹配总是选择最合适的一个, 而特化模板比通用模板在特定取值下更合适, 模板特化可以实现模板的递归编程。

```cpp
template <int N>
struct Factorial 
{
    enum { value = N * Factorial<N - 1>::value };
};
 
template <> // 特化, 当N = 1时
struct Factorial<0> 
{
    enum { value = 1 };
};
 
void foo()
{
    int x = Factorial<4>::value; // == 24
    int y = Factorial<0>::value; // == 1
}
```

#### SFINAE

SFINAE全称为Substitution failure is not an error, 表示匹配失败不是错误。这类似于重载, 模板在匹配时总是尝试匹配最精确的。我们可以基于这个特性设置模板函数或者模板类, 使具有某种性质的对象才会匹配模板, 从而起到筛选的作用。

C++的类型不一定是关键字, 利用判断是非除了`bool`关键字表示类型, 单纯的true可以用`std::true_type()`表示, false可以`std::false_type()`类型表示。对于类型C++有自己的规则可以向处理变量一样处理类型。

例如
```cpp
struct X {
  typedef int type;
};

struct Y {
  typedef int type2;
};

template <typename T> void foo(typename T::type);    // Foo0, 输入参数类型为T::type, 说明匹配的模板类T需要有T::type类型参数
template <typename T> void foo(typename T::type2);   // Foo1
template <typename T> void foo(T);                   // Foo2

void callFoo() {
   foo<X>(5);    // Foo0: Succeed, Foo1: Failed,  Foo2: Failed, 将匹配foo(typename T::type)
   foo<Y>(10);   // Foo0: Failed,  Foo1: Succeed, Foo2: Failed
   foo<int>(15); // Foo0: Failed,  Foo1: Failed,  Foo2: Succeed 只能匹配Foo2, Foo0和Foo1都匹配不了
}
```

`std::endif()`也是用这个原理进行类型的筛选, `std::decay_t`实现了类型退化, 例如将`int&, const int&, int*`等退化成`int`。

```cpp
template <typename T> void inc_counter(
  T& counterObj, 
  typename std::enable_if<is_base_of<T, ICounter>::value>::type* = nullptr );   // is_base_of<T, ICounter>::value结果为true才执行is_base_of<T, ICounter>::type, 说明模板参数T需要满足is_base_of<T, ICounter>::value结果为true, 才能顺利解析模板

template <typename T> void inc_counter(
  T& counterInt,
  typename std::enable_if<
    std::is_integral<T>::value
  >::type* = nullptr );
// 说明std::is_integral<T>::value为true, 即T为整数才能解析模板
```

如果有这样的规则, `如果类型实现了某函数,则通过模板解析; 如果没有则不通过`, 可以使用`decltype`。
```cpp
// std::decay_t是类型退化, decltype(++intTypeCounter)表示针对T变量, 如果实现了operator++()则通过解析
template <typename T>
void inc_counter(T& intTypeCounter, std::decay_t<decltype(++intTypeCounter)>* = nullptr) {
  ++intTypeCounter;
}

// decltype(counterObj.increase())表示针对T变量, 如果实现了increase()则通过解析
template <typename T>
void inc_counter(T& counterObj, std::decay_t<decltype(counterObj.increase())>* = nullptr) {
  counterObj.increase();
}
```

#### 模板元编程的一些应用

更加一般的, 检查某个类型是否含有某个函数。`std::is_same`编译期判断类型是否相等, 运行期用`typeid`。 `std::declval<U>()`返回一个类型的右值引用, 不管是否有没有默认构造函数或该类型不可以创建对象。
```cpp

#include<iostream>
#include<utility>
#include<type_traits>
 
template<typename T>
struct has_member_foo
{
private:
    template<typename U>
        static auto Check(int) -> decltype( std::declval<U>().foo(), std::true_type() );    // 第一个表达式有效, 返回std::true_type()类型
    template<typename U>
        static std::false_type Check(...);  // 直接返回std::false_type类型, 变长参数可接受任意长度参数
public:
    enum { value = std::is_same<decltype(Check<T>(0)),std::true_type>::value  };    // 调用Check, 返回类型否为为std::true_type类型
};
 
struct myStruct
{
    void foo() { std::cout << "hello" << std::endl;  }
};
 
struct another
{
    void test() { std::cout << "test" << std::endl;  }
};
 
int main()
{
    if( has_member_foo<myStruct>::value )
    	std::cout << "myStruct has foo funciton"  << std::endl;
    else
        std::cout << "myStruct does't have foo funciton"  << std::endl;
 
    if( has_member_foo<another>::value )
    	std::cout << "another has foo function"  << std::endl;
    else
	std::cout << "another does't have foo function"  << std::endl;
    return 0;
}
输出
myStruct has foo funciton
another does't have foo function
```

### 模板元编程和bind, lambda 对象

std::bind表达式的近似实现
```cpp
#pragma once
namespace tymeta {

    typedef unsigned int size_t;

    // 以下将true, false封装成编译期的class
	template<class T, T val>
	struct constant_val {
		typedef constant_val<T, val> type;
		typedef T value_type;
		constexpr operator value_type() const {
			return value;
		}
		constexpr value_type operator()() const {
			return value;
		}
		static constexpr T value = val;
	};

	typedef constant_val<bool, true> truetype;
	typedef constant_val<bool, false> falsetype;

    // 该宏定义, 实现first second字符串的粘连
    #define ty_cat(first, second) first##second

	template<class T>
	struct ty_sfinae_wrapper {
		using type = void;
	};

    // 该宏定义, 对于参数T, 如果能够解析ty_sfinae_wrapper<typename T::member_name>::type, 即T有member_name的成员, 则has_member_name为truetype, 反之has_member_name为falsetype
	#define ty_has_xxx(member_name) \
	template<class T, class U = void> \
	struct ty_cat(has_, member_name){ \
		using type = falsetype; \
		const static bool value = falsetype::value; \
	};\
	template<class T> \
	struct ty_cat(has_, member_name)<T, typename ty_sfinae_wrapper<typename T::member_name>::type>{\
		using type = truetype; \	
		const static bool value = truetype::value;\
	};\
    // 宏定义替换, 表示has_type, 即ty_cat<T> 中T是否有type这个memebr
	ty_has_xxx(type)

	// 参数, 模板参数不是类型, 而是常量。并且设置const static value = N
	template<unsigned int N>
	struct arg {
		const static unsigned int value = N;
	};

	// 通过递归的方式得到每个arg的type， N表示当前参数数量
	// 注意getArg<1>和getArg<2>是两种类型, 最终针对每个Arg实例化生成getArg<1>, getArg<2>... 等N个类型
	template<unsigned int N, class firstArg, class ...trueArgs>
	struct getArg {
		using type = typename getArg<N - 1, trueArgs...>::type;
	};

	template<class firstArg, class ...Args>
	struct getArg<1, firstArg, Args...> {
		using type = firstArg;
	};

	// 定义模板, 可变参数...trueArgs可以为0个参数
	template<class T, class ...trueArgs>
	struct lambdaImpl {
		using type = T;
	};

	// 传入N, 参数args
	template<
		unsigned int N,
		class ...trueArgs
	>
	struct lambdaImpl<arg<N>, trueArgs...>{
		// type是getArg<N>的类型, 表示arg数组
		using type = typename getArg<N, trueArgs...>::type;
	};

    // 传入到T是一个参数列表例如T<int, int>
	template<
		template<class ...Args> class T,
		class ...Args
	>
	struct lambdaImpl<T<Args... >>{ // 这里lambdaImpl接收若干template, 例如<_int<24>, _int<22>>这种列表
		struct type {

			template<class A, class U>
			struct apply_aux{};

			template<class U>
			struct apply_aux<truetype, U>{
				using type = typename U::type;
			};
			
			template<class U>
			struct apply_aux<falsetype, U>{
				using type = U;
			};


			template<class ...trueArgs>
			struct apply {
                // lambdaImpl<Args, trueArgs...>会与lambdaImpl<arg<N>, trueArgs...>, Args这里当成一个数组
				using solved_type = T< typename lambdaImpl<Args, trueArgs...>::type... >;	// 可以解析得到每个args的type tymeta::lambdaImpl<tyPlus<tymeta::arg<1>, tymeta::arg<2> > >, T这里是个列表, 列表内容是被解析完的getArg<N, trueArgs...>::type, 也就是arg<1>, arg<2>

				using type = typename apply_aux<typename has_type<solved_type>::type, solved_type>::type;	// has_type是宏定义实例化出的类, 表示solved_type里有type成员, type为solved_type::type, 如果没有返回solved_type
			};
		};
	};


	template<class T>
	struct lambda {
		using type = typename lambdaImpl<T>::type;
	};

	// arg<1> 是一个arg类型, 包括_1, _2都是类型
	namespace placeholders {
		typedef arg<1> _1;
		typedef arg<2> _2;
		typedef arg<3> _3;
		typedef arg<4> _4;
		typedef arg<5> _5;
		typedef arg<6> _6;
		typedef arg<7> _7;
		typedef arg<8> _8;
		typedef arg<9> _9;
		typedef arg<10> _10;
	}
}
```

调用
```cpp
#include <meta_test.h>
#include <iostream>

using namespace std;

// 传入两个类型参数, 该类型参数具有value值, value记录了该占位符表示第N个参数
template<class T1, class T2>
struct tyPlus {
	using type = tyPlus<T1, T2>;
    
    void print() {
        cout << T1::value<<endl;
    }
	const static int value = (T1::value + T2::value);
};

template<int val>
struct _int {
	using type = _int<val>;
	const static int value = val;
	operator int() {
		return value;
	}
};

int main() {
	// 基于占位符, 能够实现填充
    // 用placeholders::_1类型来构造tyPlus
    // 返回类型为lambdaImpl<T>::type
    using tyDouble = tymeta::lambda<tyPlus<tymeta::placeholders::_1, tymeta::placeholders::_2>>::type;
    // 相当于tymeta::lambda<tyPlus<tymeta::placeholders::_1, tymeta::placeholders::_2>>::type::apply<_int<24>, _int<22>>::type::value
    cout << tyDouble::apply<_int<24>, _int<22>>::type::value << endl;   // 实例化成tyPlus<_int<24> _int<22>>
}
```

这里`<tymeta::placeholders::_1, tymeta::placeholders::_2>`相当于`Args...`, 即`tyPlus<tymeta::placeholders::_1, tymeta::placeholders::_2>`满足`T<Args...>`。T的类型是`tyPlus`

通过`tyDouble::apply`调用`typename lambdaImpl<T<Args...>>::type::template apply<trueArgs...>::type`; 通过`T< typename lambdaImpl<Args, trueArgs...>::type... >`, 这里的`trueArgs`就是`_int<24>, _int<22>`。递归实例化出若干`getArg<1>`,`getArg<2>`的对象, 每个type为`_int<>`类型。

返回的`solved_type`是T参数列表, 因为T类型是`tyPlus`, 因为solve_type其实就是`tyPlus<_int<24>, _int<22>>`, 最后用了`using type typename apply_aux`如果T解析成功则`::type`为`tyPlus<_int<24>, _int<22>>`。最后`::value`即调用`value = _int<24>::value + _int<22>::value ` = 46。

虽然std::bind用到了placeholder::_1, 但placeholder::_1实际是一个类型
```cpp
	template<unsigned int N>
	struct arg {
		const static unsigned int value = N;
	};
```
后面的调用例如`f(2,3)`就是将该类型用参数实例化, 这就是std::bind的基本原理。例外, std::bind绑定的函数是一个地址, 或者说函数指针, 因此对于类型用`std::bind(&A::a, 1)`表示取地址。

相比之下, 我们写的lambda表达式首先会被编译期封装成一个函数对象的形式, 这和`bind`处理差不多。因为最终都会生成重载`operator()`的函数对象, 也就是functor。额或者说闭包, 某种意义上, lambda表达式和std::bind都是重载`operator()`类的语法糖。

lambda表达式优势在于快捷的写一个方法且能够作为operator()看待
```cpp
auto lambda_func = [=](int x, int y){return x + y;};
// 修改成
class lambda_1hkh1k235g12k34hklj123hl44gjk435hg6{
    operator()(int x, int y){
        return x + y;
    }
};

int x = 2;
auto lambda = [x](int y) {
    return x + y;
};
```

std::function的优势在于可以作为函数对象的接口, 类似于函数指针。`std::function<void(int ,int)>`表示它可以接收任何形参为`(int,int)`,返回值为`void`的函数对象。也就是functor, 什么是functor, 重载了`operator()`的就是。例如

```cpp
void operator()(int a, int b)
{
    Distance D;
    // 进行随机计算
    D.feet = a  + 10;
    D.inches = b + 100;
}
```
