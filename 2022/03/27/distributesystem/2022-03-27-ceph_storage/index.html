    整体Introduction

    provide **Ceph Object Storage **and/or Ceph Block Device services to Cloud Platforms, deploy a **Ceph File System **or use Ceph for another purpose, all Ceph Storage Cluster deployments begin with setting up each Ceph Node, your network, and the Ceph Storage Cluster.

    • Monitors

    Ceph Monitor (ceph-mon) maintains maps of the cluster state, including the monitor map,** manager map**, the OSD map, the MDS map, and the CRUSH map.
    状态信息对于Ceph daemons之间的通信是必须的，Ceph daemons同时维护了authentication between daemons and clients. 至少三个Moitors节点对于维护redundancy and high availability

    • Managers

    A Ceph Manager daemon (ceph-mgr) is responsible for keeping track of runtime metrics and the current state of the Ceph cluster, including storage utilization, current performance metrics, and system load. 追踪ceph节点的状态，系统指标负载等。此外还提供了ceph dashboard和rest api

    • Ceph OSDs

    An Object Storage Daemon (Ceph OSD, ceph-osd) stores data, handles data replication, recovery, rebalancing, and provides some monitoring information to Ceph Monitors and Managers by checking other Ceph OSD Daemons for a heartbeat. 存储和管理数据的基本单位

    • MDSs

    A Ceph Metadata Server (MDS, ceph-mds) stores metadata on behalf of the Ceph File System (i.e., Ceph Block Devices and Ceph Object Storage do not use MDS). 为文件系统存储元数据Ceph Metadata Servers allow POSIX file system users to execute basic commands (like ls, find, etc.) without placing an enormous burden on the Ceph Storage Cluster. 执行文件命令不需要大量调整OSD

    • CRUSH 算法

    Ceph stores data as objects within logical storage pools. 存储数据以Object形式在逻辑存储池里。
    Using the CRUSH algorithm, Ceph calculates which placement group (PG) should contain the object, and which OSD should store the placement group. 通过CRUSH算法，Ceph计算包括Object的placement group和包含placement group 的OSD,The CRUSH algorithm enables the Ceph Storage Cluster to scale, rebalance, and recover dynamically.

    handware recommendations

    • CPU

    CephFS metadata servers (MDS) are CPU-intensive. CephFS metadata servers (MDS) should therefore have quad-core (or better) CPUs and high clock rates (GHz).
    OSD nodes need enough processing power to run the RADOS service, to calculate data placement with CRUSH, to replicate data, and to maintain their own copies of the cluster map.
    Monitor nodes and manager nodes have no heavy CPU demands and require only modest processors.

    • RAM

    Generally, more RAM is better. Monitor / manager nodes for a modest cluster might do fine with 64GB; for a larger cluster with hundreds of OSDs 128GB is a reasonable target.
    Bluestore uses its own memory to cache data rather than relying on the operating system’s page cache.

    cephadm

    cephadm deploys and manages a Ceph cluster. It does this by connecting the manager daemon to hosts via SSH. The manager daemon is able to add, remove, and update Ceph containers. 一个访问工具，通过ssh连接到远程节点的daemon

    The cephadm command can

    1. bootstrap a new cluster
    2. launch a containerized shell with a working Ceph CLI
    3. aid in debugging containerized Ceph daemons

    CEPH FILE SYSTEM

    The Ceph File System, or CephFS, is a POSIX-compliant file system built on top of Ceph’s distributed object store, RADOS. 注意CephFS和RADOS不是一个东西，后者相当于是HDFS
    including traditional use-cases like shared home directories, HPC scratch space, and distributed workflow shared storage. 如共享主目录、HPC 暂存空间和分布式工作流共享存储。CephFS achieves these goals through the use of some novel architectural choices.
    file metadata is stored in a separate RADOS pool from file data and served via a resizable cluster of Metadata Servers, or MDS. 文件元数据存放在RADOS池，通过MDS提供服务
    Clients of the file system have direct access to RADOS for reading and writing file data blocks. 客户端直接访问RADOS来读写数据块For this reason, workloads may linearly scale with the size of the underlying RADOS object store因此RADOS的负载是线性增加的
    Access to data is coordinated through the cluster of MDS which serve as authorities for the state of the distributed metadata cache cooperatively maintained by clients and MDS. 对数据的访问是通过 MDS 集群来协调的，该集群充当由客户端和 MDS 共同维护的分布式元数据缓存状态

    Mutations to metadata are aggregated by each MDS into a series of efficient writes to a journal on RADOS MDS元数据的变化写入到RADOS的日志中
    Cephfs is the oldest storage interface in Ceph and was once the primary use-case for RADOS.

    客户端主要访问的是meta datasever和data server

    Metadata operations usually take up more than 50 percent of all file system operations. Also the metadata scales in a more complex fashion when compared to scaling storage (which in turn scales I/O throughput linearly). This is due to the hierarchical and interdependent nature of the file system metadata. 对元数据的操作往往占据文件系统操作的50%以上，且元数据扩展相比于存储数据扩展更加复杂，这就是文件系统元数据分层和相互依赖的的原因

    In traditional subtree partitioning, subtrees of the file system hierarchy are assigned to individual MDSs. This metadata distribution strategy provides good hierarchical locality,** linear growth of cache** and horizontal(水平的) scaling across MDSs and a fairly good distribution of metadata across MDSs.

    The problem with traditional subtree partitioning is that the workload growth by depth (across a single MDS) leads to a hotspot of activity. 问题在于某个子树随着操作造成深度增长，不均衡
